# 真男人提示词阶段生成器 使用说明书

版本日期：2026-07-09<br>
适用版本：`ComfyUI-RealMan-Prompt-Stage-Generator` 单节点内嵌版<br>
推荐环境：ComfyUI / ComfyUI Portable / Windows

## 1. 这个节点是做什么的

`真男人提示词阶段生成器` 是一个面向 ComfyUI 的提示词生成节点。它的目标不是简单拼接标签，而是把标签、随机主题、智能文本、NSFW 工作台、模型/API、参考图片和标签块编排统一到一个节点里，生成更连贯、更可控的正向提示词。

它适合这些场景：

- 想快速生成写实、商业摄影、古风、神话、插画、私房写实、电影感等不同风格提示词。
- 想通过一句自然语言，让节点自动判断主体、场景、服装、光影、镜头和画质。
- 想用本地 GGUF 或 API 模型辅助重写提示词。
- 想把标签块拖动排序，再由 Skill 或模型生成一段自然画面说明。
- 想让 NSFW 工作台里的选择真正参与全局提示词生成。
- 想保留每次生成记录，并能从历史中查看、恢复、删除提示词。

## 2. 一分钟上手

1. 打开 ComfyUI。
2. 右键添加节点，找到 `真男人提示词阶段生成器`。
3. 先保持默认参数，点击 ComfyUI 的运行按钮。
4. 在节点底部看 `终端预览`。
5. 需要一句话匹配时，切到 `智能文本`，输入例如：`雨夜街头，一个成年女性，全身人像，电影感`，点击 `匹配启用`。
6. 想随机变化时，打开 `运行时随机标签`，选择随机强度和主题池，再运行。
7. 想使用模型时，点击底部 `模型`，选择 `仅Skill`、`本地GGUF` 或 `API接口`。

最稳的默认流程是：

`选择标签` -> `可选输入智能文本` -> `运行节点` -> `查看终端预览` -> `满意后接到正向提示词输入`

## 3. 安装与分享

### 3.1 推荐安装路径

把整个文件夹放到：

```text
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator
```

不要只复制单个 `.py` 或 `web` 文件夹。前端 UI、后端节点、资源文件、依赖检查脚本需要保持在同一个节点目录中。

### 3.2 分享给别人时怎么复制

推荐把整个节点文件夹压缩后分享：

```text
ComfyUI-RealMan-Prompt-Stage-Generator/
```

对方解压到自己的：

```text
ComfyUI/custom_nodes/
```

最终目录应该长这样：

```text
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/__init__.py
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/stage_prompt_generator.py
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/web/
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/assets/
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/使用说明书.md
```

### 3.3 内嵌安装脚本

如果你使用的是便携版 ComfyUI，可以运行：

```text
内嵌安装到ComfyUI.bat
```

它用于把节点安装到 ComfyUI 的 `custom_nodes` 目录。更新已有安装时，会保留 `prompt_tag_library_custom.json` 和 `online_search_config.json`，避免覆盖用户自定义标签与搜索配置。

如果检测到旧目录 `comfyUI-qwen3_5-llama-TE`，安装器会先迁移其中缺失的用户数据，再把旧目录无损改名为以 `.disabled` 结尾的备份，防止 ComfyUI 同时加载新旧两份插件。确认新版本运行正常后，再自行归档该备份即可。

脚本会从当前目录推导 ComfyUI 路径；无法识别时会要求手动输入。路径中包含空格或中文也可以正常使用。

### 3.4 依赖检查与补装

节点主界面不应该依赖额外包才能显示。可选依赖主要影响本地模型、图片反推、OCR 或图像处理能力。

可用脚本：

```text
一键检查依赖.bat
自动补装依赖.bat
```

常见可选依赖包括：

- `llama-cpp-python`：本地 GGUF 模型调用。
- `opencv-python`：图像处理。
- `rapidocr-onnxruntime`：部分 OCR 或反推辅助。
- `Pillow` / `numpy`：图片处理基础能力。

如果双击 `.bat` 出现乱码或一堆“不是内部或外部命令”，通常是批处理编码或路径问题。优先尝试：

- 把节点目录放到没有特殊符号的路径。
- 使用 ComfyUI 自带 Python 手动安装依赖。
- 重新下载完整节点文件夹，避免脚本被不完整复制。

## 4. 节点输出口说明

节点右侧主要输出：

- `结果全文`：包含完整诊断、标签、提示词、负面词等信息。
- `首条正向提示词`：最常用，直接接到正向提示词输入。
- `已选标签`：当前参与生成的标签摘要。
- `JSON结果`：结构化结果，方便给其他节点或调试使用。
- `推荐负面词`：节点推荐的负面提示词。
- `正向提示词合集`：多条生成时的合集。
- `智能文本`：智能文本或模型处理后的结果。

一般出图只需要接：

```text
首条正向提示词 -> 正向提示词输入
推荐负面词 -> 负面提示词输入
```

## 5. 底部按钮速查

节点底部按钮是主要工作区：

- `标签`：打开标签库，选择主体、服装、场景、构图、光影、画质等；每个分类最多可选 `10` 个标签。
- `随机跑`：按当前随机策略生成变化结果。
- `匹配启用`：把输入的一句话自动匹配成标签和提示词。
- `模型`：选择 Skill、本地 GGUF 或 API 模型。
- `设定图`：生成角色设定图或多视角设定图提示词。
- `随机`：快速触发随机生成。
- `示例`：打开风格示例和推荐模板。
- `预设`：保存、加载、管理常用配置。
- `NSFW`：打开 NSFW 工作台。
- `历史`：查看、恢复、查看提示词、置顶、删除历史。
- `回退`：恢复到上一步状态。
- `槽位`：管理各类标签槽位和自定义保留项。
- `高级`：展开模型、随机、语言、输出等高级参数。
- `清空`：清空当前节点状态、预览或临时输入。

## 6. 终端预览

终端预览用于快速确认当前输出，不用每次都打开输出节点。

常用页签：

- `正向`：显示最终正向提示词。
- `负面`：显示推荐负面词。
- `标签`：显示当前被识别和使用的标签。
- `编排`：显示标签块编排结果。
- `JSON`：显示结构化结果。
- `智能文本`：显示智能文本匹配或模型处理结果。

如果运行后外接 `展示文本` 节点有内容，但终端预览仍显示“还没有正向提示词”，请先刷新浏览器页面，再重新运行一次节点。历史和终端预览都依赖前端拿到最新执行结果。

## 7. 模型按钮说明

点击 `模型` 可以选择三种生成方式。

### 7.1 仅Skill

这是默认模式。

特点：

- 不需要模型文件。
- 不需要 API Key。
- 速度最快。
- 适合随机生成、标签生成、智能文本基础匹配。

如果你只是想稳定生成提示词，优先使用 `仅Skill`。

### 7.2 本地GGUF

用于调用本地 GGUF 模型。

需要：

- 安装 `llama-cpp-python`。
- 准备 `.gguf` 模型文件。
- 在模型面板里选择模型系列、主模型、上下文长度、GPU 层数等。

常见问题：

- 报 `缺少所需模型`：说明本地模型路径没有找到或没有选择模型文件。
- 报 `Value 0 smaller than min of 256`：说明上下文长度等数值被写成了不合法值，改回 `4096`、`8192` 或更高即可。
- 本地模型加载慢是正常现象，首次运行会更慢。

### 7.3 API接口

用于调用 OpenAI 兼容接口、OpenRouter、DeepSeek、Kimi、通义、硅基流动、Claude、Gemini、Ollama、LMStudio 等服务。

常用参数：

- `API服务商`：选择服务商或兼容模式。
- `API地址`：填写接口 Base URL，不要带查询参数或 `#` 片段。
- `API密钥`：推荐填写 `env:环境变量名`。直接填写的 Key 会进入工作流数据。
- `API模型`：填写模型名。
- `API超时秒`：建议 `30` 到 `120`。
- `API额外请求头`：只有服务商需要特殊 Header 时再填。
- `额外要求`：可以写偏好，例如更写实、更商业、更简洁。

建议：

- API Key 不要分享给别人。
- 如果要公开分享工作流，先清空 API Key。
- 命名服务商使用 `env:` 密钥时，只允许请求该服务商的预设来源；工作流不能把环境变量密钥改送到其他域名。
- `OpenAI兼容` 或 `自定义` 地址需要使用环境变量密钥时，在启动 ComfyUI 前设置 `QWEN_TE_CUSTOM_API_SECRET_ORIGINS`，内容为精确来源，格式是 `scheme://host[:port]`。
- 多个可信来源可用逗号、分号或换行分隔，例如 `https://api.example.com;http://127.0.0.1:9000`。不要填写路径、查询参数或通配符。
- API 失败时节点应回退到 Skill 结果，不应该让 UI 消失。

## 8. 正向提示词生成方式

节点有几种生成方式，可以混合使用。

### 8.1 标签生成

在 `标签` 里选择内容，再运行节点。

适合：

- 你明确知道要什么主体、服装、场景、镜头。
- 想要可控、可复现的提示词。

### 8.2 智能文本

在终端预览的 `智能文本` 输入一句话，点击 `匹配启用`。

示例：

```text
成年女性，雨夜街头，商业摄影，全身人像，冷色霓虹
```

节点会自动判断：

- 主体是谁。
- 是人物、非人物还是场景。
- 更偏文本优先还是节点标签优先。
- 是否需要使用 NSFW 工作台。
- 应该走哪条风格轨道。

智能文本不是把标签简单拼起来，而是尽量生成一段连贯的正向画面说明。

### 8.3 随机运行时

打开 `运行时随机标签` 后，每次运行都会按当前随机策略变化。

适合：

- 批量找灵感。
- 让人物、服装、环境、光影、动作、道具每次有所区别。
- 避免所有风格都长得一样。

推荐配合：

- `随机主题池：自动`
- `随机强度：中` 或 `强`
- `智能文本风格优先：自动判断`

### 8.4 标签块编排

点击终端预览里的 `编排`，可以把当前标签导入成标签块。

你可以：

- 拖动标签块顺序。
- 删除不想要的块。
- 插入文字块。
- 把标签块和自然语言混合。
- 点击 `生成提示词`，让 Skill 或模型按你的顺序重写。

它适合精修提示词阅读顺序，比如：

```text
画面风格 -> 主体 -> 服装 -> 场景 -> 构图 -> 光影 -> 动作 -> 画质
```

如果你调用 API 或本地模型，标签块编排会优先把块顺序交给模型理解；如果模型不可用，会使用 Skill 兜底生成。

### 8.5 参考图片与图片反推

节点支持 `参考图片` 输入。

用途：

- 参考单人图生成设定图。
- 根据图片反推主体、服装、色彩、场景方向。
- 配合 `设定图` 按同一角色生成多视角提示词。

注意：

- 不启用设定图时，参考图片不应该强行污染普通提示词。
- 启用设定图时，才会按设定图 Skill 生成角色展示类提示词。

## 9. 语言输出

`提示词语言` 支持：

- `纯中文`
- `英文提示词+中文说明`
- `纯英文`

建议：

- 中文模型或中文工作流用 `纯中文`。
- 想兼容更多绘图模型时用 `纯英文`。
- 想自己检查含义时用 `英文提示词+中文说明`。

节点会尽量保证：

- 中文输出不是夹杂半截英文标签。
- 英文输出不是中文直译堆砌。
- 中英混合模式中，英文用于提示词，中文用于解释或补充。

## 10. 随机主题池与模板风格

### 10.1 模板风格

模板风格决定整体媒介方向，例如：

- 真实感
- 商业摄影
- 古风电影
- 神话感
- 插画感
- 私房写实
- CG / 游戏感
- 复古电影感

模板风格不是锁死。它只是告诉节点这次更应该往哪条主轨道收敛。

### 10.2 随机主题池

随机主题池决定每次随机从哪类题材中取素材，例如：

- 自动
- 写实生活
- 糖水写真
- 都市职场
- 夜场电影感
- 古风园林
- 神话史诗
- 赛博工业
- 复古插画

如果你想每次差异更大：

- 使用 `自动`。
- 提高随机强度。
- 不要锁太多核心标签。
- 清理不必要的固定标签白名单。

如果你想稳定在某个方向：

- 选择具体主题池。
- 降低随机强度。
- 增加核心标签锁定数量。

### 10.3 核心标签锁定怎么用

`核心标签锁定数量` 只影响 `运行时随机标签`，不会影响普通手动生成、普通智能文本生成或普通 API 重写。

它的真实作用是：在随机生成前，最多保留当前已选标签里的前 N 个核心标签，然后再让随机补齐其他槽位。

使用逻辑：

- `运行时随机模式 = 保留已选核心标签`：严格按 `核心标签锁定数量` 保留当前核心标签。
- `运行时随机模式 = 全随机`：不保留当前已选标签，只按主题池重新随机。
- `运行时随机模式 = 重写主体与场景`：保留风格、光影、构图、动作等，重新换主体和场景。
- `运行时随机模式 = 自动判断`：少量主体、构图、场景锚点会自动保留；标签很多时会更倾向重写主线或全随机，避免看起来像锁死。
- `锁定标签白名单`：优先级最高，写进去的标签会尽量强制保留；如果槽位满了，会放到自定义补充里。
- `随机排除标签`：用于禁止随机抽到某些标签；但白名单里的标签不会被排除。

推荐用法：

- 想固定人物和全身构图：选择 `保留已选核心标签`，锁定数量设为 `2-5`。
- 想只固定少数关键词：把关键词写进 `锁定标签白名单`。
- 想每次变化明显：选择 `全随机` 或把锁定数量设为 `0`。
- 想换人物和环境但保留风格：选择 `重写主体与场景`。
- 不确定怎么选：使用 `自动判断`。

## 11. 风格隔离与反锁死

节点会做两件事：

- `风格隔离`：防止古风、赛博、私房、商业摄影等风格互相串词。
- `反锁死`：防止每次都生成同一套人物、场景、光影和动作。

你看到的 `解析：...` 会告诉你本次判断结果，例如：

```text
解析：智能 自动判断 -> 文本优先 / 商业摄影 / 全身人像
```

这能帮助你判断：

- 当前是节点标签优先还是文本优先。
- 随机模式是否重写主线。
- NSFW 工作台是否参与。
- 模型/API 是否参与。

如果提示词总是重复，优先检查：

- 是否锁定了太多核心标签。
- 是否随机强度过低。
- 是否把同一批标签写进了白名单。
- 是否智能文本输入过于固定。
- 是否 API 的系统提示词覆盖写得太死。

## 12. NSFW 工作台

NSFW 工作台用于管理合法成年内容方向的标签和负面预设。它不是只给智能文本用，而是可以作用到全局生成链路。

### 12.1 启用方式

1. 点击底部 `NSFW`。
2. 在工作台中选择结构、镜头、负面、预设等内容。
3. 点击 `启用工作台`。
4. 点击 `写回当前节点` 或 `写回并同步负面`。
5. 回到节点外部运行。

工作台启用后，外部 `NSFW` 按钮应保持启用状态。运行时会进入 NSFW 分支，并把工作台选择写入全局上下文。

### 12.2 NSFW 自定义字段库

NSFW 工作台内有独立的 `自定义字段库`。

用途：

- 给 NSFW 下拉字段单独添加自己的选项。
- 不污染主标签库。
- 只在 NSFW 工作台内使用。

使用方式：

1. 选择字段，例如 `场景`、`动作`、`服饰`、`情绪`。
2. 在输入框中一行写一个自定义选项。
3. 点击 `加入当前字段`。
4. 回到对应下拉框即可选择。

自定义字段库和主标签库是分开的。你在 NSFW 里加的内容不会自动加入普通标签库。

### 12.3 NSFW 与智能文本

当 NSFW 工作台启用时：

- 智能文本会自动倾向 `成人向成熟` 策略。
- 会优先读取 NSFW 工作台已选择的标签。
- 会补充成年主体和稳定构图护栏。
- 仍然会尊重语言输出、随机策略和模型设置。

注意：请只用于合法成年角色和合规内容。节点会尽量避免把高风险年龄、强迫、暴力羞辱等内容作为成人向素材进入生成链路。

## 13. 设定图功能

点击 `设定图` 可以生成角色设定图提示词。

两种模式：

- `参考单人图`：接入参考图片，按图中角色生成设定图。
- `纯提示词`：不接图片，只根据当前文本和标签生成设定图。

按钮逻辑：

- `启用`：运行时会生成设定图方向提示词。
- `停用`：运行时不改写普通正向提示词。

设定图启用时，适合生成：

- 正面全身
- 侧面全身
- 背面全身
- 头像特写
- 服装和材质说明

设定图停用时，不应该强行生成三视图或多视角。

## 14. 历史功能

点击 `历史` 可以查看最近生成记录。

支持：

- `查看`：查看本次生成的完整提示词。
- `恢复`：把历史状态恢复到当前节点。
- `置顶`：把重要记录放到前面。
- `删除`：删除单条记录。
- `一键删除历史`：清空所有历史。

历史里应该显示每次生成的提示词，而不是只显示标签摘要。如果某条历史没有提示词，节点会尽量从 `首条正向提示词`、`结果全文` 或 `JSON结果` 中回填。

## 15. 预设与示例

### 15.1 预设

预设用于保存常用配置。

适合保存：

- 常用模板风格。
- 常用随机策略。
- 常用语言输出。
- 常用模型/API 配置。
- 常用负面词和质量要求。

建议不要把 API Key 放进公开分享的预设里。

### 15.2 示例

示例用于快速套用风格方向。

适合：

- 找灵感。
- 快速切换摄影、古风、神话、插画、商业、私房等方向。
- 查看推荐采样、CFG、Denoise 等建议。

示例不是锁死模板。应用后仍可以通过标签、随机、智能文本继续修改。

## 16. 高级参数怎么调

一般用户不需要经常改高级参数。

常用项：

- `提示词语言`：控制中文、英文、中英混合。
- `详细度`：控制提示词长短。
- `输出模式`：控制完整结果或提示词优先。
- `智能文本风格优先`：自动判断、节点优先、文本优先。
- `运行时随机标签`：是否启用随机。
- `运行时随机模式`：保留核心标签或重写主线。
- `核心标签锁定数量`：只在运行时随机中生效，表示最多保留多少个当前核心标签。
- `随机主题池`：控制随机题材来源。
- `锁定标签白名单`：强制保留指定标签。
- `随机排除标签`：避免某些标签出现。
- `系统提示词覆盖`：给模型/API 的高级覆盖指令。

如果你感觉提示词被锁死，优先减少：

- 核心标签锁定数量。
- 锁定标签白名单。
- 固定模板风格。
- 系统提示词覆盖里的强约束。

## 17. 清空按钮说明

`清空` 用于清理当前临时输入和预览状态。

它应该清理：

- 智能文本输入。
- 终端预览。
- 标签块编排临时内容。
- 模型面板的额外要求。
- 当前临时匹配状态。

它不应该误删：

- 已保存预设。
- 已安装模型。
- 历史记录，除非你在历史面板点击 `一键删除历史`。
- NSFW 自定义字段库，除非你在对应库里手动清理。

## 18. 常见问题

### 18.1 复制给别人后 UI 不显示

优先检查：

- 是否复制了整个节点文件夹。
- `web` 文件夹是否还在。
- `__init__.py` 是否还在。
- 浏览器是否需要强制刷新。
- ComfyUI 启动日志是否有节点加载报错。

如果只显示一堆普通参数，没有底部面板，通常是前端 JS 没有加载。

### 18.2 ComfyUI 打不开 8188

如果浏览器访问：

```text
http://127.0.0.1:8188
```

出现连接失败，先确认 ComfyUI 是否真的监听端口。

Windows 可检查：

```powershell
netstat -ano | findstr 8188
```

如果没有监听，说明不是节点 UI 问题，而是 ComfyUI 服务没有正常启动。

### 18.3 本地模型报缺失

检查：

- 是否选择了 `.gguf` 文件。
- 模型文件是否还在原路径。
- `llama-cpp-python` 是否安装成功。
- 上下文长度是否大于等于 `256`。

如果只是想用节点，不想折腾模型，切回 `仅Skill`。

### 18.4 API 生成重复、像锁死

检查：

- API 面板的 `额外要求` 是否写了固定内容。
- `系统提示词覆盖` 是否太强。
- 智能文本是否每次输入一样。
- 随机强度是否太低。
- 是否启用了太多锁定标签。
- API 模型本身是否倾向复读。

可尝试：

- 清空额外要求。
- 使用 `自动判断`。
- 提高随机强度。
- 改为 `文本优先`。
- 换 API 模型或提高温度。

### 18.5 NSFW 里选了标签但外面没生效

需要确认：

- 工作台是否已启用。
- 是否点击了 `写回当前节点`。
- 是否点击了 `写回并同步负面`。
- 外部 NSFW 按钮是否显示启用状态。
- 历史或终端预览里是否显示 NSFW 分支参与。

### 18.6 历史查看没有提示词

先重新运行一次节点。历史记录会优先保存最终正向提示词，如果旧记录只有标签摘要，可以删除旧记录后重新生成。

### 18.7 终端预览不显示提示词

可能原因：

- 节点还没运行。
- 前端缓存旧版本。
- API 结果还没写回前端。
- 当前查看的是 `JSON` 或 `智能文本` 页签而不是 `正向`。

处理方式：

- 运行节点一次。
- 刷新浏览器。
- 切换到 `正向` 页签。
- 打开 `历史` 查看是否有记录。

### 18.8 古风、商业、赛博互相串词

减少串词的方法：

- 每次只保留一个主风格轨道。
- 清理旧标签。
- 不要把多个强风格同时写入自定义补充。
- 随机模式选择 `重写主线` 时，允许节点替换旧场景和旧服装。

### 18.9 生成内容总是粉色、总是全身、总是某个物品

检查是否存在：

- `玫瑰粉调`、`高饱和` 等固定光影标签。
- `全景全身`、`人物完整入镜` 被锁定。
- 某个物品在白名单或自定义补充里。
- API 额外要求里写死了同一描述。

## 19. 推荐工作流

### 19.1 稳定写实人像

1. 模板风格选 `真实感` 或 `商业摄影`。
2. 构图选 `全景全身` 或 `中景半身`。
3. 光影选择一到两个，不要太多。
4. 运行一次，看终端预览。
5. 不满意再用智能文本补一句方向。

### 19.2 批量随机找灵感

1. 打开 `运行时随机标签`。
2. 随机主题池选 `自动`。
3. 随机强度选 `中` 或 `强`。
4. 核心标签锁定数量不要太高。
5. 每次运行后到历史里查看和收藏。

### 19.3 智能文本精修

1. 在智能文本输入一句自然描述。
2. 点击 `匹配启用`。
3. 查看解析结果。
4. 如果偏离你的输入，切 `文本优先`。
5. 如果想保留节点标签，切 `节点优先`。

### 19.4 API 精修长提示词

1. 模型来源选 `API接口`。
2. 选择服务商，并优先填写 `env:环境变量名`；自定义地址按上面的可信来源规则配置。
3. 详细度选择 `详细`。
4. 输入智能文本或导入标签块编排。
5. 运行后检查终端预览和历史查看。

### 19.5 NSFW 工作台

1. 打开 `NSFW`。
2. 选择结构、镜头、负面和质量档位。
3. 可在 `自定义字段库` 添加自己的字段选项。
4. 点击 `启用工作台`。
5. 点击 `写回当前节点`。
6. 回到外面运行节点。

## 20. 给别人安装前的检查清单

分享前建议确认：

- 已清空直接填写的 API Key，或确认工作流里只保留安全的 `env:变量名` 引用。
- 节点文件夹完整。
- `web` 文件夹存在。
- `使用说明书.md` 存在。
- `一键检查依赖.bat` 和 `自动补装依赖.bat` 存在。
- 在自己的 ComfyUI 里能看到底部 UI 面板。
- 运行一次能输出 `首条正向提示词`。
- 历史里能查看本次生成提示词。
- NSFW 工作台启用后外部按钮状态正常。

## 21. 最短口诀

新手用：

```text
仅Skill + 标签 + 智能文本 + 终端预览
```

想更智能：

```text
API接口 + 智能文本 + 标签块编排
```

想更随机：

```text
运行时随机 + 自动主题池 + 中/强随机
```

想更可控：

```text
锁核心标签 + 降低随机强度 + 使用预设
```

想排错：

```text
看终端预览 -> 看历史 -> 看 ComfyUI 日志 -> 检查依赖
```

这个节点的核心思路是：标签负责素材，Skill 负责判断，模型/API 负责重写，终端预览和历史负责确认结果。只要这四件事能串起来，提示词就会越来越稳。
