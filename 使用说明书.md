# 真男人提示词阶段生成器 使用说明书

版本日期：2026-07-22<br>
适用版本：`ComfyUI-RealMan-Prompt-Stage-Generator` 单节点内嵌版<br>
推荐环境：ComfyUI / ComfyUI Portable / Windows

## 内嵌网页浏览器性能与链接

- 网页画面采用交互后短时快速刷新、空闲降频策略；交互帧保持原生显示尺寸，稳定帧按设备像素比高清采样。检测到可见视频或连续变化画面时自动使用低开销动态帧并提高刷新频率，iframe 播放器和 Canvas 动画也能自动降载；暂停或静止后恢复高清静态帧。轮询周期会扣除截图耗时，避免处理时间与固定等待叠加。
- 拖动与滚轮事件采用最新值合并，避免慢请求在输入队列中堆积；切回页面会立即恢复刷新。
- 搜索结果中的 `_blank` 链接和 `window.open` 会留在当前内嵌页面打开，地址栏与前进/后退状态随页面跳转更新。
- 点击坐标按实际截图尺寸和 `object-fit: contain` 留白换算；弹窗、窗口或显示缩放变化时，后端网页视口会自动同步。截图坐标会跟随当前滚动位置，长页面滚动后不会误抓顶部而变成白屏或只显示细条。
- 前端异常刷新或崩溃导致关闭请求未送达时，空闲的内嵌浏览器会话也会在 20 分钟后自动结束，避免视频或动画长期占用后台资源。

## 节点加载与槽位性能

- 节点添加和工作流加载不再为 200 个隐藏标签槽位重复传输完整标签目录，旧工作流中的标签值保持兼容。
- `槽位` 面板第一次展开时才创建内容。每个分组的输入共用一份候选列表，可直接输入关键词筛选；不在库中的内容请写入面板底部的 `自定义补充`。
- 标签、模型、预设、历史等面板会复用已验证的标签库快照；标签增删后自动失效并重新校验，未变化的响应使用压缩和 `ETag` 减少传输。

## Danbooru 通用视觉标签与生成收敛

节点内置一组经过筛选的通用视觉标签，覆盖构图、镜头、光影、色彩、场景、动作、服装和媒介风格。仅使用通用视觉词，不导入画师、版权、角色、元数据、幼态或露骨性标签。

这些标签会参与所有模式：普通标签生成、运行时随机、随机主题池、模板风格、智能文本、标签块编排、角色设定图、图片反推、NSFW 工作台，以及本地/API 模型后置。后端会在生成前统一收敛互斥视点、景别、背景、色彩方案、动作和主光方向；参考设定表会自动切换到简洁背景，并移除鱼眼、第一人称等不适合设定表的动态镜头。

普通标签库当前包含 2568 个内置标签。除原有丁达尔光、体积光、天气、时段、山地、水域和风景构图外，现已补充光源互动、天空天象、地质奇观、植被生态、人文风景、天文地景、建筑风景、叙事构图和单帧稳定构图。新增内容仍按小类隔离，选择风景不会自动把人物、NSFW、古风、赛博或其他主题池素材混进同一画面。

## 1. 这个节点是做什么的

`真男人提示词阶段生成器` 是一个面向 ComfyUI 的提示词生成节点。它的目标不是简单拼接标签，而是把标签、随机主题、智能文本、NSFW 工作台、模型/API、参考图片和标签块编排统一到一个节点里，生成更连贯、更可控的正向提示词。

它适合这些场景：

- 想快速生成写实、商业摄影、古风、神话、插画、私房写实、电影感等不同风格提示词。
- 想通过一句自然语言，让节点自动判断主体、场景、服装、光影、镜头和画质。
- 想用本地模型、内置 GGUF 或 API 模型辅助重写提示词。
- 想把标签块拖动排序，再由 Skill 或模型生成一段自然画面说明。
- 想让 NSFW 工作台里的选择真正参与全局提示词生成。
- 想保留每次生成记录，并能从历史中查看、恢复、删除提示词。

## 2. 一分钟上手

1. 打开 ComfyUI。
2. 右键添加节点，找到 `真男人提示词阶段生成器`。
3. 先保持默认参数，点击 ComfyUI 的运行按钮。
4. 在节点底部看 `终端预览`。
5. 需要一句话匹配时，切到 `智能文本`，输入例如：`雨夜街头，一个成年女性，全身人像，电影感`，点击 `匹配启用`。
6. 想随机变化时，打开 `运行时随机标签`，选择随机强度和主题池，再运行。
7. 想使用模型时，点击底部 `模型`，选择 `仅Skill`、`本地模型` 或 `API接口`。旧工作流里的 `本地GGUF` 会自动迁移为 `本地模型`。
8. 想访问网页或找新的画面词时，点击终端标题栏的 `浏览器`；使用 `网页浏览器` 或 `提示词搜索` 页签。

最稳的默认流程是：

`选择标签` -> `可选输入智能文本` -> `运行节点` -> `查看终端预览` -> `满意后接到正向提示词输入`

## 3. 安装与分享

### 3.1 推荐安装路径

把整个文件夹放到：

```text
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator
```

不要只复制单个 `.py` 或 `web` 文件夹。前端 UI、后端节点、资源文件、依赖检查脚本需要保持在同一个节点目录中。

同一个 ComfyUI 的 `custom_nodes` 中只能保留一份启用的本插件。GitHub ZIP 常见目录名是 `ComfyUI-RealMan-Prompt-Stage-Generator-main`，它可以直接加载，但不要同时再保留标准目录、旧版 `comfyUI-qwen3_5-llama-TE`、手工复制目录或另一个 Manager 安装，否则两套后端和前端可能交叉加载。

### 3.2 分享给别人时怎么复制

推荐把整个节点文件夹压缩后分享：

```text
ComfyUI-RealMan-Prompt-Stage-Generator/
```

对方解压到自己的：

```text
ComfyUI/custom_nodes/
```

最终目录应该长这样：

```text
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/__init__.py
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/stage_prompt_generator.py
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/web/
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/assets/
ComfyUI/custom_nodes/ComfyUI-RealMan-Prompt-Stage-Generator/使用说明书.md
```

### 3.3 内嵌安装脚本

如果你使用的是便携版 ComfyUI，可以运行：

```text
内嵌安装到ComfyUI.bat
```

它用于把节点安装到 ComfyUI 的 `custom_nodes` 目录。更新已有安装时，会保留 `prompt_tag_library_custom.json` 和 `online_search_config.json`，避免覆盖用户自定义标签与搜索配置。

如果检测到旧目录 `comfyUI-qwen3_5-llama-TE`，安装器会先迁移其中缺失的用户数据，再把旧目录无损改名为以 `.disabled` 结尾的备份，防止 ComfyUI 同时加载新旧两份插件。确认新版本运行正常后，再自行归档该备份即可。

脚本会从当前目录推导 ComfyUI 路径；无法识别时会要求手动输入。路径中包含空格或中文也可以正常使用。

### 3.4 依赖检查与补装

节点主界面不应该依赖额外包才能显示。可选依赖主要影响本地模型、图片反推、OCR 或图像处理能力。

可用脚本：

```text
一键检查依赖.bat
自动补装依赖.bat
```

常见可选依赖包括：

- `llama-cpp-python`：节点内置 GGUF 路线；外接兼容模型和本地 API 服务不强制依赖它。
- `opencv-python`：图像处理。
- `rapidocr-onnxruntime`：部分 OCR 或反推辅助。
- `Pillow` / `numpy`：图片处理基础能力。

如果双击 `.bat` 出现乱码或一堆“不是内部或外部命令”，通常是批处理编码或路径问题。优先尝试：

- 把节点目录放到没有特殊符号的路径。
- 使用 ComfyUI 自带 Python 手动安装依赖。
- 重新下载完整节点文件夹，避免脚本被不完整复制。

## 4. 节点输出口说明

节点右侧主要输出：

- `结果全文`：包含完整诊断、标签、提示词、负面词等信息。
- `首条正向提示词`：最常用，直接接到正向提示词输入。
- `已选标签`：当前参与生成的标签摘要。
- `JSON结果`：结构化结果，方便给其他节点或调试使用。
- `推荐负面词`：节点推荐的负面提示词。
- `正向提示词合集`：多条生成时的合集。
- `智能文本`：智能文本或模型处理后的结果。
- `视频提示词`：独立视频 Skill 生成的单镜头自然语言提示词；配置本地模型或 API 时，会在 Skill 底稿上进行视频专用后置润色。

一般出图只需要接：

```text
首条正向提示词 -> 正向提示词输入
推荐负面词 -> 负面提示词输入
视频提示词 -> 视频生成节点的提示词输入
```

## 5. 底部按钮速查

节点底部按钮是主要工作区：

- `标签`：打开标签库，选择主体、服装、场景、构图、光影、画质等；每个分类最多可选 `20` 个标签。面板会显示 `已选/上限`，到达上限后仍可点击已选标签取消，再继续选择其他标签。
- `随机跑`：按当前随机策略生成变化结果。
- `匹配启用`：把输入的一句话自动匹配成标签和提示词。
- `模型`：选择 Skill、本地模型或 API；本地模型可使用外接兼容对象或内置 GGUF。
- `设定图`：生成角色设定图或多视角设定图提示词。
- `随机`：快速触发随机生成。
- `示例`：打开风格示例和推荐模板。
- `预设`：保存、加载、管理常用配置。
- `NSFW`：打开 NSFW 工作台。
- `历史`：查看、恢复、查看提示词、置顶、删除历史。
- `回退`：恢复到上一步状态。
- `槽位`：管理各类标签槽位和自定义保留项。
- `高级`：展开模型、随机、语言、输出等高级参数。
- `清空`：清空当前节点状态、预览或临时输入。

终端标题栏还有 `浏览器` 入口。它包含网页浏览和提示词搜索两个页签，不会恢复旧版的第二套按钮 UI，也不会在节点运行时自动联网。

## 6. 终端预览

终端预览用于快速确认当前输出，不用每次都打开输出节点。

常用页签：

- `正向`：显示最终正向提示词。
- `负面`：显示推荐负面词。
- `标签`：显示当前被识别和使用的标签。
- `编排`：显示标签块编排结果。
- `JSON`：显示结构化结果。
- `智能文本`：显示智能文本匹配或模型处理结果。
- `视频`：显示视频 Skill 或视频模型后置润色后的提示词；没有运行结果时会显示等待提示。

如果运行后外接 `展示文本` 节点有内容，但终端预览仍显示“还没有正向提示词”，请先刷新浏览器页面，再重新运行一次节点。历史和终端预览都依赖前端拿到最新执行结果。

### 6.1 网页浏览器与提示词搜索

1. 点击终端标题栏的 `浏览器`。弹窗默认显示本地网页主页，不会自动访问外部网站。
2. 在 `网页浏览器` 页签输入完整网址、域名或网页搜索词，点击 `打开`。
3. 等待本机 Edge/Chromium 会话启动后，节点会显示实时网页画面；可直接点击链接和按钮、滚动页面、输入文字、粘贴内容，并使用返回、前进和刷新。
4. 画面按当前显示区域等比缩放，稳定后自动切换为适配设备像素比的高清帧；视频、iframe 播放器或 Canvas 连续动画期间自动切换为动态帧模式，以较低编码开销换取更高刷新率，暂停或静止后恢复高清。点击坐标会换算到真实网页，窗口尺寸改变时后端视口自动同步。切换到 `提示词搜索` 时暂停画面轮询，切回后立即继续。
5. 如果系统策略不允许 Edge/Chromium 远程调试，内嵌预览会自动切换到真正的独立浏览器窗口，并同步当前网址；之后点击 `打开` 会继续使用独立浏览器模式。登录、下载、证书检查、文件选择器或复杂弹窗也可使用右上角 `独立浏览器`；地址栏右侧箭头可在普通新标签页打开当前网址。
6. 切换到 `提示词搜索` 页签，输入主题、风格、人物或场景关键词并点击 `搜索`；只有这一步会把提示词关键词发送到已配置来源。
7. 检查完整提示词和右侧预览，可按来源、优选程度和提示词长度筛选。
8. 使用 `回填提示词` 将所选完整提示词加入当前节点，也可复制提示词或执行拆分入库。

内嵌主视图不再使用 iframe，而是由后端启动隔离的本机 Edge/Chrome/Chromium 无头会话并回传 JPEG 画面，所以不受网站 `X-Frame-Options` 或 CSP iframe 限制。鼠标、滚轮、键盘、粘贴和中文输入会通过本机受保护接口转发到真实浏览器页面。

内嵌会话使用临时独立配置，默认禁止下载；关闭弹窗或返回本地主页时会关闭会话并清理临时目录。内嵌启动失败时会显示明确原因，并自动调用 `ComfyUI/user/companion_browser/profile` 下的独立持久配置打开可见 Edge/Chromium。该窗口拥有完整地址栏、标签页、登录、下载和证书信息，关闭节点弹窗不会同时关闭它。

地址栏和后端会拒绝危险协议、账号密码、当前 ComfyUI 页面以及直接输入的本机/内网地址，请求还要求本机同源与专用请求头。这只是入口保护，不是网络隔离；网页打开后仍可能自行跳转或发起其他网络请求。

Windows 会自动查找 Microsoft Edge、Google Chrome 或 Chromium。若使用便携 Chromium，可在启动 ComfyUI 前设置 `QWEN_TE_BROWSER_EXE` 为可信可执行文件的绝对路径；文件名必须是 `msedge.exe`、`chrome.exe` 或 `chromium.exe`。更新插件后需要完整重启 ComfyUI，不能只刷新网页。

搜索结果不会自动写入提示词，也不会随每次 ComfyUI 队列运行重复联网。外部来源不可用时，节点会自动回退到本地标签库联想。

如需通用网页搜索，可在 `online_search_config.json` 中把 `searxng_base_url` 设置为你自己的 HTTPS SearXNG 实例；也可以在启动 ComfyUI 前通过环境变量 `QWEN_TE_SEARXNG_URL` 覆盖。实例地址只从本机配置读取，工作流和前端请求不能指定任意搜索服务器。SearXNG 一旦返回有效结果，就不会继续访问 Civitai 或 Lexica。

`public_source_fallback_enabled` 控制公开来源回退。设为 `false` 后，SearXNG 失败、无结果或未配置时会直接使用本地标签库，不再请求 Civitai/Lexica；设为 `true` 则保留公开来源回退。公开仓库建议使用环境变量保存私有实例地址，避免误提交内部域名。

相同关键词会复用短时缓存的原始搜索样本，但“已在库/待新增”状态会根据当前标签库重新计算。成功搜索缓存约 3 分钟；全源失败只缓存约 20 秒，避免离线时连续重复等待。

该实例必须启用 JSON 输出（`format=json`）。当前不支持纯 HTTP localhost、URL 内账号密码或自定义认证头；自签名/企业证书可通过 `SSL_CERT_FILE` 或 `REQUESTS_CA_BUNDLE` 指定 CA 证书包。

## 7. 模型按钮说明

点击 `模型` 可以选择三种生成方式。

### 7.1 仅Skill

这是默认模式。

特点：

- 不需要模型文件。
- 不需要 API Key。
- 速度最快。
- 适合随机生成、标签生成、智能文本基础匹配。

如果你只是想稳定生成提示词，优先使用 `仅Skill`。

视频输出在此模式下也会正常生成。它不调用模型，直接使用视频 Skill 形成一条单地点、单主运镜的连续自然语言剧情；中文正文稳定保持 `800-1200` 字，并包含触发、动作、环境反馈、声音和最终定格，正文不写具体秒数或时长参数。

### 7.2 本地模型

`本地模型`不是只指 GGUF，目前支持三条路线：

- 在主节点的 `qwen模型` 输入或“通用模型阶段式提示词生成器”的 `通用模型` 输入连接兼容模型对象。对象可直接提供 `create_chat_completion`、`invoke`、`generate_content`、`complete`、`predict`、`chat` 或可调用文本接口，也可以包在常见的 `llm/model/client/pipeline` 容器里。
- 不连接模型输入时，使用节点内置 GGUF 路线：安装 `llama-cpp-python`，把 `.gguf` 放入 `ComfyUI/models/LLM`，再选择模型系列、主模型、上下文长度和 GPU 层数。
- Ollama、LM Studio 或其他本机 HTTP 服务从 `API接口` 进入，选择对应本地服务预设或填写 OpenAI 兼容地址；这类路线不需要节点直接加载 GGUF。

模型返回的 `<think>`、`<analysis>`、Markdown 标题、JSON/消息包装会先清洗，并兼容多 `choice`、Responses API 嵌套输出和对象式 SDK 响应；只有分析字段而没有最终正文时不会误把分析当成提示词。图像/智能文本模型的短草稿会融入 Skill 剧情骨架；视频输出则单独调用视频后置润色链路，并重新检查 800-1200 字、单镜头因果、语言和主体/场景/动作锚点。首次空文本会按瞬时响应自动重试一次，首次正文只有分析、占位符或不合格内容时还会发送一次简化正文修复请求；仍无可用正文才回退到对应 Skill 底稿。

常见问题：

- 报 `缺少所需模型`：内置路线没有找到或没有选择模型文件；也可以改为连接外接兼容模型。
- 报 `Value 0 smaller than min`：说明旧工作流把上下文等数值写成了非法值；当前上下文合法范围从 `1024` 开始，普通任务改回 `4096` 或 `8192`。
- 本地模型加载慢是正常现象，首次运行会更慢。

### 7.3 API接口

用于调用 OpenAI 兼容接口、OpenRouter、DeepSeek、Kimi、通义、硅基流动、Claude、Gemini、Ollama、LM Studio 等服务。

常用参数：

- `模型来源`：必须明确切换为 `API接口`；如果仍是 `仅Skill`，下面即使填写了地址或 Key 也不会发请求。
- `API服务商`：选择服务商或兼容模式。切换到不同服务商会写入其预设地址、模型和 `env:` 密钥引用，并清空旧的额外请求头。
- `API地址`：只接受 `http://` 或 `https://` Base URL，不得包含用户名、密码、空白、查询参数、片段、反斜杠、转义点段、中文路径或控制字符。OpenAI 兼容接口会自动补 `/chat/completions`。
- `API密钥`：推荐填写 `env:环境变量名`。`env:变量名` 通常只读取指定变量；旧默认 `env:QWEN_TE_API_KEY` 为空时会继续尝试当前服务商的标准环境变量。直接填写的 Key 会进入工作流数据。
- `API模型`：填写模型名；带预设模型的服务商允许输入框留空，由后端使用当前预设。
- `API超时秒`：默认 `120`，长提示词建议 `90` 到 `180`。超时、断连、429 和服务端 5xx 会自动有界重试一次；401/403 等认证错误不会重试。
- `API额外请求头`：支持 JSON 对象，或每行 `名称: 值` / `名称=值`。格式错误、重复名称、空值和控制字符会直接报错。禁止覆盖 `Authorization`、`Proxy-Authorization`、`X-API-Key`、`X-Goog-API-Key`、`Content-Type`、`Accept-Encoding`、`Host`、`Content-Length`、`Transfer-Encoding`、`Connection`、`Cookie` 和 `Set-Cookie`。
- `额外要求`：可以写偏好，例如更写实、更商业、更简洁。

当前主要预设：

- OpenAI：`gpt-4o-mini`
- OpenRouter：`openai/gpt-4o-mini`
- DeepSeek：`https://api.deepseek.com` + `deepseek-v4-flash`
- 通义：`qwen-plus`
- Kimi：`moonshot-v1-8k`
- 智谱：`glm-4.5`
- Groq：`openai/gpt-oss-20b`
- Mistral：`mistral-small-latest`
- Perplexity：`sonar`
- Gemini：`gemini-2.5-flash`
- Claude：`claude-haiku-4-5`
- Ollama：`qwen2.5`

`OpenAI兼容`、SiliconFlow、火山方舟、Together、Fireworks、LM Studio 和 `自定义` 没有固定默认模型时，必须手动填写。服务商可能调整模型生命周期，升级插件后应以模型面板中的当前预设为准。

运行和安全说明：

- 模型面板顶部会显示 `API 未启用`、`等待运行`、`调用成功`、`部分回退`、`调用已恢复` 或 `调用失败`。不要只看配置摘要判断 API 是否实际工作。
- `model_active_fallback_count` 表示最终仍使用 Skill 的输出条数；`model_call_failure_count` 是逻辑调用失败次数。瞬时传输错误重试成功时会在纠偏说明中显示“已恢复”，活动回退仍为零。
- JSON 还提供 `model_call_attempt_count`、`model_call_adopted_count`、`model_fallback_note` 和 `model_config_signature`。前端用配置签名避免把上一次地址、模型、Key 或 Header 的结果误当成当前状态；签名不包含认证明文。
- 命名远程服务商在预设来源缺少 Key 时会在联网前明确回退并显示缺少的环境变量名。
- `OpenAI兼容`、`自定义` 和本地接口在 Key 留空时不会自动借用通用环境变量。
- 命名服务商使用 `env:` 密钥时，只允许请求该服务商的预设来源；工作流不能把环境变量密钥改送到其他域名。
- 自定义地址需要使用环境变量密钥时，在启动 ComfyUI 前设置 `QWEN_TE_CUSTOM_API_SECRET_ORIGINS`，内容为精确来源 `scheme://host[:port]`。多个来源可用逗号、分号或换行分隔，不要填写路径、查询参数或通配符。
- API 失败时节点会回退到 Skill 结果，并把 401、重试后仍超时、错误模型名、空响应或响应校验失败写入模型面板、标签摘要和 JSON。错误信息会脱敏。
- 视频模型调用会复用同一份本地/API连接和超时重试策略，但使用独立的视频系统提示词；`视频提示词模型状态` 会显示“未调用（仅Skill）”“已采用模型润色”“模型调用成功，保留 Skill 结果”或失败回退原因。
- JSON 中的 `video_prompt_model_status`、`video_prompt_model_source` 和 `video_prompt_required_anchor_count` 用于确认视频是否实际经过模型，以及本次保护了多少个事实锚点。模型失败不会清空第 8 个输出口。
- OpenAI 的 o1/o3/o4/GPT-5 模型会自动使用 `developer` 消息和 `max_completion_tokens`，并省略不兼容的采样参数；DeepSeek V4 会关闭 thinking；Claude/Gemini 原生接口会映射 `top_k` 和停止序列，Gemini 还会映射 `seed`。
- 批量响应没有按节点分隔格式返回时，8 条以内会自动逐条重试；更大的批量会保留 Skill 结果并显示回退原因，避免意外发出大量 API 请求。批量首次调用与逐条调用共用同一份模式合同，不会因为生成数量变化而丢失随机、编排、设定图或 NSFW 上下文。
- API Key 不要分享给别人。公开工作流前应清空直接填写的 Key。

### 7.4 视频提示词的模型润色链路

视频提示词不是把图像标签直接改成长句，而是两阶段生成：

1. 视频 Skill 先读取主体、服装、场景、动作、道具、光影和全局创作主线，生成一条 `800-1200` 字的单镜头自然语言底稿。
2. 选择 `本地模型` 或 `API接口` 后，节点把这条底稿交给同一个模型连接，用视频专用系统提示词进行连贯化和细节润色。
3. 模型候选必须保留主体、场景、动作等事实锚点，并通过语言、长度、因果链、单主运镜和反分析文本检查，才会替换 Skill 底稿。
4. 候选不合格、响应为空、接口超时、认证失败或本地模型不支持文本调用时，节点保留原 Skill 结果，并在模型状态和 JSON 中记录原因。

因此，`模型来源：本地模型` 不再等同于“只调用 GGUF”：它既可以使用外接兼容对象，也可以使用节点内置 GGUF；`API接口` 也可以连接云端服务、Ollama 或 LM Studio。两种路径都会影响视频第 8 个输出口，但不会改变前 7 个输出口的顺序。

## 8. 正向提示词生成方式

节点有几种生成方式，可以混合使用。所有模式会先按统一优先级合并：用户显式输入与锁定标签优先，其次是角色设定图/图片反推可见事实、标签块顺序与锁定块、NSFW 保护锚点，再到运行时随机、主题池和模板档案，最后才允许模型补充未明确的细节。主题池负责题材、身份和地点主线，模板档案负责媒介、光影与成片质感；场景、服装、动作、光影和道具默认各收敛为一条主方向，技术画质最多保留两条互补方向，避免两个档案互相串词。最终连续避重也会识别当前模式：主提示词、智能文本和视频分别记录历史；视频只改变跟随方式、环境反馈和声音节奏，不追加图像专用的“画面变化方向”，也不写具体秒数。非人物不会被追加人物腿部或发丝动作，角色设定图会保留多视角结构，编排会保留锁定块顺序，NSFW 会保留成年主体和工作台保护锚点。

### 8.1 标签生成

在 `标签` 里选择内容，再运行节点。

每个分组的 20 个槽位由标签弹窗、槽位面板、预设载入、NSFW 映射和运行时状态共同使用。旧工作流的前 10 个槽位名称不变，新增槽位会自动填入“无”；如果导入的预设超过某个分组容量，超出部分会进入“自定义补充”，不会静默丢失。随机模式不会因为扩容而自动填满新增槽位，自动补充仍以每组前 10 个槽位作为预算，避免提示词变成标签堆叠。

适合：

- 你明确知道要什么主体、服装、场景、镜头。
- 想要可控、可复现的提示词。

### 8.2 智能文本

在终端预览的 `智能文本` 输入一句话，点击 `匹配启用`。

示例：

```text
成年女性，雨夜街头，商业摄影，全身人像，冷色霓虹
```

节点会自动判断：

- 主体是谁。
- 是人物、非人物还是场景。
- 更偏文本优先还是节点标签优先。
- 是否需要使用 NSFW 工作台。
- 应该走哪条风格轨道。

智能文本不是把标签简单拼起来，而是尽量生成一段连贯的正向画面说明。

### 8.3 随机运行时

打开 `运行时随机标签` 后，每次运行都会按当前随机策略变化。

适合：

- 批量找灵感。
- 让人物、服装、环境、光影、动作、道具每次有所区别。
- 避免所有风格都长得一样。

推荐配合：

- `随机主题池：自动`
- `随机强度：中` 或 `强`
- `智能文本风格优先：自动判断`

### 8.4 标签块编排

点击终端预览里的 `编排`，可以把当前标签导入成标签块。

你可以：

- 拖动标签块顺序。
- 删除不想要的块。
- 插入文字块。
- 把标签块和自然语言混合。
- 点击 `生成提示词`，让 Skill 或模型按你的顺序重写。

它适合精修提示词阅读顺序，比如：

```text
画面风格 -> 主体 -> 服装 -> 场景 -> 构图 -> 光影 -> 动作 -> 画质
```

如果你调用 API 或本地模型，标签块编排会优先把块顺序交给模型理解；如果模型不可用，会使用 Skill 兜底生成。

### 8.5 参考图片与图片反推

节点支持 `参考图片` 输入。

用途：

- 参考单人图生成设定图。
- 根据图片反推主体、服装、色彩、场景方向。
- 配合 `设定图` 按同一角色生成多视角提示词。

注意：

- 不启用设定图时，参考图片不应该强行污染普通提示词。
- 启用设定图时，才会按设定图 Skill 生成角色展示类提示词。

## 9. 语言输出

`提示词语言` 支持：

- `纯中文`
- `英文提示词+中文说明`
- `纯英文`

建议：

- 想兼容更多绘图模型时用 `纯英文`。

### 10.3 单帧、人数与多视图规则

- 普通人物：单张连续画面、同一决定性时刻、同一人物只出现一次、一个清晰头部和一张脸。
- 显式双人/群像：保留用户选择的人数，所有人同处一张图；每人只出现一次，脸部、站位和肢体关系分别可读。
- 角色设定图/三视图/四联画/显式镜中视图：允许多视图，但同一身份、服装和脸部特征要一致；不会额外添加重复角度或叙事时间切片。
- 非人物主体：主主体只出现一次并保持稳定轮廓，不会随机补人脸或把不同时间状态拼进一张图。
- 推荐负面词会按这四种情况自动切换。普通单图加入分屏、拼贴、故事板、堆叠肖像、重复脸、额外头部和克隆主体等负向词；设定图则改用跨视图身份/服装一致性负向词，不会错误禁止用户主动选择的多视图。

这是全局自动判断，不需要新增面板参数。用户显式选择优先：选了双人、情侣、伴侣、队伍或群像标签就不会被压成单人；启用设定图或选择角色三视图就不会被普通单帧规则破坏。普通单图不要在 `额外要求` 中写“分别展示开端、经过、结尾”“上下三段”“先推近再拉远”等时间序列指令。

简洁模式的内容优先级是：用户显式选择和工作台字段优先，预设补充其次，通用质量词最后。运行时档案替换出的服装、动作或光影如果不属于原始槽位，也会进入自然语言补充句，不会静默丢失。

如果你想每次差异更大：

- 使用 `自动`。
- 提高随机强度。
- 不要锁太多核心标签。
- 清理不必要的固定标签白名单。

如果你想稳定在某个方向：

- 选择具体主题池。
- 降低随机强度。
- 增加核心标签锁定数量。

### 10.4 核心标签锁定怎么用

`核心标签锁定数量` 只影响 `运行时随机标签`，不会影响普通手动生成、普通智能文本生成或普通 API 重写。

它的真实作用是：在随机生成前，最多保留当前已选标签里的前 N 个核心标签，然后再让随机补齐其他槽位。

使用逻辑：

- `运行时随机模式 = 保留已选核心标签`：严格按 `核心标签锁定数量` 保留当前核心标签。
- `运行时随机模式 = 全随机`：不保留当前已选标签，只按主题池重新随机。
- `运行时随机模式 = 重写主体与场景`：保留风格、光影、构图、动作等，重新换主体和场景。
- `运行时随机模式 = 自动判断`：少量主体、构图、场景锚点会自动保留；标签很多时会更倾向 `重写主体与场景` 或全随机，避免看起来像锁死。
- `锁定标签白名单`：优先级最高，写进去的标签会尽量强制保留；如果槽位满了，会放到自定义补充里。
- `随机排除标签`：用于禁止随机抽到某些标签；但白名单里的标签不会被排除。

推荐用法：

- 想固定人物和全身构图：选择 `保留已选核心标签`，锁定数量设为 `2-5`。
- 想只固定少数关键词：把关键词写进 `锁定标签白名单`。
- 想每次变化明显：选择 `全随机` 或把锁定数量设为 `0`。
- 想换人物和环境但保留风格：选择 `重写主体与场景`。
- 不确定怎么选：使用 `自动判断`。

## 11. 风格隔离与反锁死

节点会做两件事：

- `风格隔离`：防止古风、赛博、私房、商业摄影等风格互相串词。
- `反锁死`：防止每次都生成同一套人物、场景、光影和动作。

两者通过同一份全局创作主线配合：隔离只负责本次运行内部收敛，不会把模板永久写死；反锁死会在下一次运行重新选择剧情弧和未锁定维度。模型、本地 GGUF 和 API 处于优先级末端，只能补主线空白，不能把已经确定的雨夜站台改成古代宫殿，或把写实摄影改成水彩、CG 等另一条媒介。调试时可在节点 JSON 输出查看 `global_creative_spine` 和 `global_creative_spine_summary`。

你看到的 `解析：...` 会告诉你本次判断结果，例如：

```text
解析：智能 自动判断 -> 文本优先 / 商业摄影 / 全身人像
```

这能帮助你判断：

- 当前是节点标签优先还是文本优先。
- 随机模式是否使用 `重写主体与场景`。
- NSFW 工作台是否参与。
- 模型/API 是否参与。

如果提示词总是重复，优先检查：

- 是否锁定了太多核心标签。
- 是否随机强度过低。
- 是否把同一批标签写进了白名单。
- 是否智能文本输入过于固定。
- 是否 API 的系统提示词覆盖写得太死。

## 12. NSFW 工作台

NSFW 工作台用于管理合法成年内容方向的标签和负面预设。它不是只给智能文本用，而是可以作用到全局生成链路。

### 12.1 启用方式

2. 在工作台中选择结构、镜头、负面、预设等内容。
3. 点击 `启用工作台`。
4. 点击 `写回当前节点` 或 `写回并同步负面`。

工作台启用后，外部 `NSFW` 按钮应保持启用状态。运行时会进入 NSFW 分支，并把工作台选择写入全局上下文。

当前内置目录包含 26 个结构化字段、915 个选项条目、16 套预设和 7 套负面方案。常用字段覆盖成年身份与关系、服装、动作、场景、表情、道具、情绪、剧情方式、镜头、焦段、光源、丁达尔/体积光、色调、视觉风格、效果和滤镜。新增成年职业、服装、关系动作、电影放映室/观星套房/雨夜花园等场景、双人同焦平面、镜面主次、丁达尔光束落点和材质反射控制都使用纯正向描述，并遵守合法成年、单帧身份稳定与人物不复制规则。

映射优先级为：手动字段与选择器 > 预设填充 > 质量层级。预设只填空字段，不覆盖已经手动选好的值。器官与显式行为始终是主动选择项，不会被 `全部随机` 自动抽取；全部随机只处理场景、动作、服装、情绪和影像控制字段。

### 12.2 NSFW 自定义字段库

NSFW 工作台内有独立的 `自定义字段库`。

用途：

- 给 NSFW 下拉字段单独添加自己的选项。
- 不污染主标签库。
- 只在 NSFW 工作台内使用。

使用方式：

### 12.3 NSFW 与智能文本

当 NSFW 工作台启用时：

- 智能文本会自动倾向 `成人向成熟` 策略。
- 会优先读取 NSFW 工作台已选择的标签。
- 会补充成年主体和稳定构图护栏。

## 13. 设定图功能

点击 `设定图` 可以生成比例均衡的角色设定图提示词。默认版式严格使用三幅全身主视图：从左到右依次为正面 0 度、标准侧面 90 度、背面 180 度。

两种模式：

- `参考单人图`：接入参考图片，按图中角色生成设定图。
- `纯提示词`：不接图片，只根据当前文本和标签生成设定图。

按钮逻辑：

- `启用`：运行时会生成设定图方向提示词。
- `停用`：运行时不改写普通正向提示词。

设定图启用时，适合生成：

- 正面、标准侧面、背面三幅等宽全身视图，列宽比例固定为 `1:1:1`。
- 三幅人物使用相同高度、同一头顶线和脚底基线、统一镜头高度与正交投影。
- 三幅视图保持中性自然站姿，从头顶到鞋底完整入镜，角色身份、体型、发型、服装、配色和材质一致。
- 头像、表情或服装材质细节默认不加入；只有在额外要求中明确提出时，才放入独立辅助带，三幅主视图尺寸和等宽比例保持不变。
- 场景标签只提供简洁连续背景的配色与材质线索，不会与动态动作、鱼眼、第一人称或具体叙事场景混在一起。

使用建议：优先选择横向宽画幅，单次生成 `1` 张并关闭运行时随机；参考图尽量使用无遮挡的清晰单人全身图。需要四视图、局部细节表或其他布局时，在额外要求中明确写出视图数量和顺序，节点会保留显式布局并平均分配画幅。

设定图停用时，不应该强行生成三视图或多视角。

## 14. 历史功能

点击 `历史` 可以查看最近生成记录。

支持：

- `查看`：查看本次生成的完整提示词。
- `恢复`：把历史状态恢复到当前节点。
- `置顶`：把重要记录放到前面。
- `删除`：删除单条记录。
- `一键删除历史`：清空所有历史。

历史里应该显示每次生成的提示词，而不是只显示标签摘要。如果某条历史没有提示词，节点会尽量从 `首条正向提示词`、`结果全文` 或 `JSON结果` 中回填。

## 16. 参数设置完整手册

### 16.1 默认配置与调整顺序

节点默认是 `仅Skill / 生成 3 条 / 纯中文 / 标准 / 完整结果 / 自动判断随机模式 / 平衡收敛`。本地模型默认上下文 `8192`、GPU 层数 `-1`；API 默认超时 `120` 秒；模型采样默认 `1800 token / 温度 0.62 / top_p 0.9 / top_k 40 / 重复惩罚 1.08`。

推荐按下面顺序调整，不要一开始同时改变所有采样值：

1. 先确定 `模型来源`、`生成数量`、`提示词语言` 和 `输出模式`。
2. 再确定主体、模板风格、主题池与智能文本优先级。
3. 需要批量差异时才开启运行时随机，并调整锁定数量与随机强度。
4. 只有本地/API 模型出现复读、截断或变化不足时，再调整 token 与采样参数。
5. 最后保存成预设；预设会保存生成、随机、图片反推、本地模型和采样配置，但不会保存 API 密钥、额外认证头或运行缓存。

| 参数 | 默认值 | 可选值/范围 | 推荐设置 | 作用与代价 |
| --- | --- | --- | --- | --- |

| `核心标签锁定数量` | `10` | `0-500` | 明显变化用 `0-3`；稳定角色用 `4-10` | 只限制保留核心模式；白名单与 NSFW 保护标签不受该数截断 |

`seed` 不是唯一的变化来源。连续生成避重会读取最近提示词指纹，剧情规划也会轮换事件、情绪转折、空间动线和镜头结尾。因此需要严格复现时，应同时固定工作流状态、seed 和历史/避重缓存；只想让每次不同则保留 `seed=0`。

| `智能文本匹配` | 关闭 | 开/关 | 输入了自然语言描述时开启 | 开启后文本参与主体、风格、场景与剧情识别 |
| `智能文本输入` | 空 | 多行文本 | 写清“谁、为何、做什么、在哪里、发生什么变化” | 因果描述比关键词堆叠更容易得到清晰剧情；过长且自相矛盾会降低一致性 |
| `智能文本风格优先` | `自动判断` | 自动、节点优先、文本优先 | 日常自动；保风格用节点优先；忠于文案用文本优先 | 决定文本中的风格词和节点模板冲突时谁优先 |
| `标签块编排启用` | 关闭 | 开/关 | 需要固定阅读顺序时开启 | 锁定块作为高优先级事实；块过多会限制模型自由度 |
| `标签块编排JSON` | 空 | 合法编排 JSON | 通过编排界面维护 | 手工截断或无效 JSON 会被清理，不应粘贴普通提示词到这里 |
| `图片反推生成` | 关闭 | 开/关 | 接参考图且需要反推时开启 | 需要支持视觉的本地模型、API 或外接模型；仅 Skill 无法读取图片像素 |
| `图片反推模式` | `角色设定图` | `角色设定图`、`仅反推描述` | 普通反推用仅描述；均衡三视图用设定图 | 默认加入正面 0 度、标准侧面 90 度、背面 180 度三幅等宽全身主视图；细节视图仅按显式要求进入独立辅助带 |
| `图片反推最大边长` | `960` | `256-2048`，步长 64 | 快速 `640-768`；平衡 `960`；细节 `1280` | 越大视觉编码越慢、占用越高；不是越大越准确 |
| `参考图片` | 未连接 | IMAGE 输入 | 反推时连接清晰单图 | 只有图片反推/设定图启用后才进入视觉链路，避免污染普通提示词 |

### 16.5 本地模型与内置 GGUF 参数

| 参数 | 默认值 | 范围/选项 | 推荐设置 | 作用与风险 |
| --- | --- | --- | --- | --- |
| `内置模型系列` | `Qwen3.5-VL` | Qwen、Gemma、Llama、Mistral、DeepSeek、通用 GGUF | 与模型架构一致；不确定用通用 GGUF | 错误系列可能导致聊天模板或视觉 handler 不匹配 |
| `内置主模型` | 模型目录首项 | `models/LLM` 中的模型文件 | 优先使用指令微调 `.gguf` | 模型量化越大质量通常越高，但加载与推理更慢 |
| `内置视觉投影mmproj` | `无` | `无` 或匹配的 mmproj | 纯文本选无；图片反推必须选匹配文件 | 不匹配会报错或得到无效视觉结果 |
| `内置启用思考` | 关闭 | 开/关 | 提示词生成通常关闭 | 开启会增加首 token 等待、上下文占用和清洗成本 |
| `内置上下文长度` | `8192` | `1024-327680`，步长 256 | 普通提示词 `4096-8192`；长图文 `8192-16384` | 上下文越大 KV 缓存越占内存/显存；设到模型不支持的长度也不会提高质量 |
| `内置GPU层数` | `-1` | `-1-9999` | `-1` 自动尽量上 GPU；`0` 纯 CPU；OOM 时逐步降低 | GPU 层越多通常越快，但会增加显存占用 |
| `内置KV缓存K类型` | `默认(F16)` | F16、`q8_0` | 稳定优先 F16；显存紧张尝试 q8_0 | q8 需要当前 llama-cpp-python 支持，可能轻微影响精度 |
| `内置KV缓存V类型` | `默认(F16)` | F16、`q8_0` | 与 K 类型保持一致 | 混用前先确认运行库支持；报 type_k/type_v 错误时恢复 F16 |

选择本地模型后，同一个模型会依次参与主提示词、启用时的智能文本以及视频提示词后置润色。视频链路会多产生一次文本调用；使用较慢的 GGUF 时，生成数量建议先设为 `1`，上下文保持 `4096-8192`，确认 `视频提示词模型状态` 为“已采用模型润色”后再增加批量数量。

硬件建议只作为起点，实际占用取决于模型参数量、量化、上下文、mmproj 和其他 ComfyUI 模型：

| 硬件 | 模型起点 | 上下文 | GPU 层数 | KV 缓存 | 生成建议 |
| --- | --- | --- | --- | --- | --- |
| 纯 CPU | 3B-8B、Q4 左右 | `4096-8192` | `0` | F16；内存紧张试 q8 | 数量 `1`，token `1280-1800` |
| 6-8 GB 显存 | 7B-8B、Q4 左右 | `8192` | 先 `-1`，OOM 再降低 | q8 可节省 KV | 数量 `1-2`，图片边长 `640-960` |
| 12-16 GB 显存 | 7B-14B 量化 | `8192-16384` | `-1` | F16 或 q8 | 数量 `1-3`，图片边长 `960` |
| 24 GB 及以上 | 14B-32B 量化起步 | `16384-32768` | `-1` | 优先 F16 | 长图文与 `2-4` 条批量；仍应给绘图模型预留显存 |

本地模型 OOM 时的调整顺序是：先降低图片反推边长，再降低上下文长度，再切 KV 为 q8，最后降低 GPU 层数或换更小量化。不要首先把上下文拉到 327680。

### 16.6 API 参数

| 参数 | 默认值 | 范围/格式 | 推荐设置 | 作用与风险 |
| --- | --- | --- | --- | --- |
| `API服务商` | `OpenAI兼容` | 节点服务商目录 | 有明确服务商就选对应预设 | 服务商切换会更新地址、模型和 env 引用，并清除旧额外头 |
| `API地址` | 空/服务商预设 | 规范 Base URL | 优先用服务商预设；自定义再手填 | 禁止账号密码、查询参数、片段和非规范路径 |
| `API密钥` | `env:QWEN_TE_API_KEY` | 明文或 `env:变量名` | 始终推荐 env 引用 | 明文会进入工作流；命名服务商的 env 密钥只允许发送到预设来源 |
| `API模型` | 空/服务商预设 | 模型 ID | 使用服务商当前有效模型名 | 模型下线或名称错误会在调用前后回退到 Skill |
| `API超时秒` | `120` | `5-600` | 快模型 `60-120`；长提示词 `120-180` | 超时、断连、429、5xx 自动重试一次；认证与普通 4xx 不重试 |
| `API额外请求头` | 空 | JSON 或每行 Header | 只填服务商明确要求的非认证头 | 认证、Cookie、Host、Content-Type 等受保护字段不能覆盖 |

当状态显示“模型来源：API接口，模型实际来源：仅Skill回退”时，说明配置选择了 API，但最终提示词仍由 Skill 提供。结合 `模型调用失败次数`、`模型传输重试次数`、`模型最近瞬时错误` 和错误原因判断：超时先把 `API超时秒` 调到 `120-180` 并确认网络；401/403 检查 Key；404 检查 Base URL 与模型名；节点会自动重试首次空响应并兼容常见嵌套格式，如果两次仍为空或只有分析字段，再检查服务端响应格式并更换兼容模型。若原因包含“改变了当前创作主线”，说明模型返回了长度合格但与本次媒介、场景或事实锚点冲突的内容；此时应降低温度、保持默认系统提示词覆盖，并检查模板、主题池、智能文本和手动标签是否自相矛盾。回退不会丢失提示词，只是本次没有采纳模型结果。

本地模型状态同样区分两类问题：请求、加载或调用异常才计入“调用失败”；模型已经返回但最终正文不合格时显示“调用成功但输出未采用”。正常的思考包装清洗和短草稿融合不会再显示 `模型实际来源：仅Skill回退`。内置 Qwen3.5 的纯中文图像/视频后置不会再要求 4B 模型重写整份 800-1200 字底稿，而是请求 2-4 句具体增量并融合回 Skill 骨架；每条实际输出预算最多 640 token，批量按条数扩展。其他本地模型若仍回退，请先关闭模型自己的思考输出、把 `最大生成token` 调到 `1800-2600`，并检查外接对象是否提供受支持的文本调用接口。

### 16.7 模型采样参数

以下参数只直接影响本地模型、API 或外接模型；仅 Skill 的剧情变化由模板档案、全局剧情规划、seed 和连续避重控制。

| 参数 | 默认值 | 范围 | 推荐设置 | 作用与副作用 |
| --- | --- | --- | --- | --- |
| `最大生成token` | `1800` | `128-8192` | API/完整长文 `1800-2600`；内置 Qwen3.5 中文增量保持默认即可 | 太低会截断完整长文候选；内置 Qwen3.5 中文增量每条自动封顶 640，批量按数量扩展；其他批量请求总上限 8192 |
| `温度` | `0.62` | `0-2` | 稳定 `0.45-0.65`；多样 `0.72-0.9` | 太低僵硬，太高可能偏题或破坏格式 |
| `top_p` | `0.9` | `0-1` | `0.85-0.95` | 降低更保守，提高更多样；不要与温度同时大幅提高 |
| `top_k` | `40` | `0-200` | 稳定 `20-40`；多样 `40-80`；`0` 交给接口 | 某些 API 不支持会自动省略；过高可能引入低概率措辞 |
| `重复惩罚` | `1.08` | `0.5-2` | `1.05-1.15` | 太高会破坏角色名、材质名和句子连贯性 |
| `频率惩罚` | `0` | `0-2` | 复读时 `0.1-0.3` | 按出现次数抑制重复；太高会让自然语言生硬 |
| `存在惩罚` | `0` | `0-2` | 扩展细节时 `0.05-0.2` | 鼓励新词；太高会让内容不断换主题 |
| `seed` | `0` | 64 位无符号整数 | 变化用 0；复现用正整数 | 部分 API 不支持 seed，会自动忽略 |
| `输出think块` | 关闭 | 开/关 | 日常关闭 | 开启只适合诊断，会增加输出与历史体积，并可能干扰提示词阅读 |
| `系统提示词覆盖` | 内置剧情合同 | 多行文本 | 保持默认 | 默认已经约束自然语言、剧情、语言、长度与用户事实；强改容易导致锁死或格式校验回退 |

批量、运行时随机或已有历史指纹时，模型润色器会自动启用多样性下限，避免用户把参数设得过低后所有结果仍完全相同。当前下限包括温度约 `0.72`、重复惩罚 `1.1`、频率惩罚 `0.18`、存在惩罚 `0.1`；这是模型润色保护，不会改写节点面板里的保存值。

### 16.8 参数联动规则

- `温度 × top_p × top_k`：三者都扩大候选范围。先只改温度；仍不够再把 top_p 提到 `0.93-0.95`，最后才提高 top_k。三者同时拉满通常只会降低稳定性。
- `重复惩罚 × 频率惩罚 × 存在惩罚`：重复惩罚处理局部复读，频率惩罚按次数降权，存在惩罚鼓励新内容。优先小幅加频率惩罚，不要直接把三项都设成 1 以上。
- `生成数量 × 最大生成token`：普通批量模型请求会使用 `min(8192, 每条token上限 × 条数)` 的总预算；内置 Qwen3.5 中文增量使用 `min(8192, 640 × 条数, 用户上限 × 条数)`。数量越大越容易触顶；长文批量建议 2-4 条，更多数量可拆成多轮。
- `上下文长度 × 图片边长 × 显存`：上下文主要增加 KV 缓存，图片边长增加视觉 token 和编码成本，两者同时提高最容易 OOM。
- `seed × 历史避重`：正数 seed 固定随机序列，但最近提示词指纹和历史状态仍参与连续差异；严格复现需要同时恢复同一历史状态。
- `锁定数量 × 白名单 × 随机强度`：白名单优先级最高。随机看起来没变化时，先减少白名单，再把锁定数量降到 `0-3`，最后提高强度。
- `模板风格 × 风格隔离 × 主题池`：模板决定媒介，隔离策略清理冲突，主题池补题材。系列图建议具体模板 + 严格隔离 + 具体主题；探索建议自动模板 + 平衡收敛 + 自动主题。
- `全局创作主线 × 模型后置`：归一化后的主体、服装、场景、动作和主媒介先冻结，再交给 Skill、智能文本、本地 GGUF/兼容模型与 API 共用。模型可以增加材质、空间反馈、光线迁移和因果细节，但删除已有锚点或引入其他媒介/冲突场景时会被融合或回退。
- `视频 Skill × 模型后置`：视频 Skill 永远先生成可用底稿；本地模型/API只能在同一地点、同一主体和同一动作因果链内润色。候选缺少任一显式锚点、超出长度、变成多镜头或含分析文本时，第 8 个输出自动保留 Skill 底稿。
- `详细度 × token`：Skill 不受 token 限制；模型输出详细模式时 token 太低可能被截断。详细中文建议至少 `1800`，批量时注意总上限。
- `剧情长度 × 单帧合同`：800-1200 字用于补充因果、空间、材质和光线，不等于多帧。模型若删除当前人数或单帧/多视图合同，结果会被融合回 Skill 骨架或回退，不会直接采用。

### 16.10 预设、历史、浏览器和搜索配置

预设最多保留 30 项，单项上限 160 KiB、总上限 1 MiB；历史最多 50 项，使用相同的单项与总容量保护。超过容量时前端会拒绝或跳过超大项目，不会无界占用浏览器内存。历史保存生成提示词、必要参数和恢复状态；连续运行输出还会通过后端小型执行快照补齐，避免批量结果只留下最后一条。

新保存的预设会包含：风格、主体、随机策略、智能文本、标签块、图片反推、本地模型选项、API 服务商/地址/模型/超时、系统提示词和全部采样参数。为防止泄露与污染，以下内容明确不进入预设：`API密钥`、`API额外请求头`、随机/连续避重缓存和运行时预览令牌。完整 ComfyUI 工作流仍可能保存原始 API 密钥控件，所以公开工作流前必须清空明文 Key。

| 浏览器/搜索设置 | 默认值 | 怎么设置 |
| --- | --- | --- |
| `QWEN_TE_BROWSER_EXE` | 自动查找 Edge/Chrome/Chromium | 便携浏览器时设为可信 exe 的绝对路径，重启 ComfyUI |
| `searxng_base_url` | 空 | 在 `online_search_config.json` 填自己的 HTTPS SearXNG；不要覆盖整个配置文件 |
| `searxng_timeout_seconds` | `8` | 慢实例可调 `8-20`；过大会让搜索失败等待更久 |
| `public_source_fallback_enabled` | `true` | 不希望访问 Civitai/Lexica 时设 `false`，失败后直接本地回退 |
| `QWEN_TE_SEARXNG_URL` | 空 | 启动前临时覆盖 SearXNG 地址，适合不把私有域名写入仓库 |

如果你感觉提示词被锁死，按顺序检查：白名单是否过多、核心锁定是否过高、模板与主题是否都固定、系统提示词覆盖是否强约束、智能文本是否每次完全相同、API 模型是否复读。优先恢复默认采样，再逐项提高随机强度或温度，避免同时修改十几个参数后无法定位原因。

## 17. 清空按钮说明

`清空` 用于清理当前临时输入和预览状态。

它应该清理：

- 智能文本输入。
- 终端预览。
- 标签块编排临时内容。
- 模型面板的额外要求。
- 当前临时匹配状态。

它不应该误删：

- 已保存预设。
- 已安装模型。
- 历史记录，除非你在历史面板点击 `一键删除历史`。
- NSFW 自定义字段库，除非你在对应库里手动清理。

## 18. 常见问题

### 18.1 复制给别人后 UI 不显示

优先检查：

- 是否复制了整个节点文件夹。
- `web` 文件夹是否还在。
- `__init__.py` 是否还在。
- 浏览器是否需要强制刷新。
- ComfyUI 启动日志是否有节点加载报错。
- `custom_nodes` 中是否同时存在标准目录、`-main` ZIP 目录、旧目录或其他复制目录。

当前版本不会仅凭“主脚本已加载”就关闭备用界面。全新文件名的 `00_stage_prompt_generator_preflight_v1.js` 会最先收起原生标签和高级参数，并把尚未接管的旧工作流节点高度限制在安全范围；这也避免浏览器继续复用旧兼容脚本缓存。只有某个节点真正创建出完整主面板后，才会移除该节点的 `TE MINI CONSOLE`；主面板初始化失败或仍在重试时，备用工具栏会收起所有原始标签槽位和高级参数，保留标签、随机、入队、槽位、高级、NSFW 与清空等基本操作。没有 `addDOMWidget` 的旧版或替代前端会自动使用普通画布按钮，不再直接退出并留下超长节点。兼容栏会立即接管通过 `addWidget` 新增的控件，并执行有界短时复核，所以不同 ComfyUI 前端分批插入的第 11-20 个槽位也会继续被接管；后端同时把非日常输入声明为 ComfyUI 原生高级项。主面板后续接管成功时，预压缩状态和兼容按钮会被安全清理。

兼容栏诊断探针默认关闭，不会在启动扫描期间产生额外 HTTP 请求。只有需要收集前端接管日志时，才在启动 ComfyUI 前设置 `QWEN_TE_FRONTEND_PROBE_LOG=1`，并给地址加入 `?qwen_te_probe=1`；排查完成后移除参数。主界面等待 ComfyUI 前端注册的轮询同样有时间上限，极端加载失败时不会永久占用定时器。

如果显示 `TE MINI CONSOLE`，说明完整主面板没有完成接管，节点正在使用兼容界面；这时仍出现截图中那样的一长串 `主体标签11-20`、`画面风格标签11-20`，说明使用的是尚未支持后插槽位重整的旧版前端。如果连 `TE MINI CONSOLE` 也没有，才表示两套界面都没有取得控制权。更新到最新版后按以下顺序处理：

1. 关闭所有 ComfyUI 后端窗口，不要只刷新网页。
2. 在 `custom_nodes` 中只保留一份本插件；旧目录先改成以 `.disabled` 结尾，确认后再归档。
3. 确认插件的 `web` 中同时存在 `00_stage_prompt_generator_preflight_v1.js`、`stage_prompt_generator_ui_v2.js` 和 `stage_prompt_generator_mini_toolbar.js`。
4. 重新启动 ComfyUI，等待节点加载完成，再在浏览器按 `Ctrl+F5`。
5. 仍不兼容时，在地址后加入 `?qwen_te_mini=1` 强制使用备用模式，例如 `http://127.0.0.1:8188/?qwen_te_mini=1`。

标准目录、旧目录和 GitHub ZIP 常见的 `-main` / `-master` 目录名都提供主界面与备用工具栏的无缓存脚本响应。安装更新后仍必须完整重启，因为 Python 路由不会靠浏览器刷新重新注册。

### 18.2 ComfyUI 打不开 8188

如果浏览器访问：

```text
http://127.0.0.1:8188
```

出现连接失败，先确认 ComfyUI 是否真的监听端口。

Windows 可检查：

```powershell
netstat -ano | findstr 8188
```

如果没有监听，说明不是节点 UI 问题，而是 ComfyUI 服务没有正常启动。

### 18.3 本地模型报缺失

检查：

- 是否选择了 `.gguf` 文件。
- 模型文件是否还在原路径。
- `llama-cpp-python` 是否安装成功。
- 上下文长度是否大于等于 `1024`。

如果只是想用节点，不想折腾模型，切回 `仅Skill`。

### 18.4 API 生成重复、像锁死

检查：

- API 面板的 `额外要求` 是否写了固定内容。
- `系统提示词覆盖` 是否太强。
- 智能文本是否每次输入一样。
- 随机强度是否太低。
- 是否启用了太多锁定标签。
- API 模型本身是否倾向复读。

可尝试：

- 清空额外要求。
- 使用 `自动判断`。
- 提高随机强度。
- 改为 `文本优先`。
- 换 API 模型或提高温度。

### 18.5 NSFW 里选了标签但外面没生效

需要确认：

- 工作台是否已启用。
- 是否点击了 `写回当前节点`。
- 是否点击了 `写回并同步负面`。
- 外部 NSFW 按钮是否显示启用状态。
- 历史或终端预览里是否显示 NSFW 分支参与。

### 18.6 历史查看没有提示词

先重新运行一次节点。历史记录会优先保存最终正向提示词，如果旧记录只有标签摘要，可以删除旧记录后重新生成。

### 18.7 终端预览不显示提示词

可能原因：

- 节点还没运行。
- 前端缓存旧版本。
- API 结果还没写回前端。
- 当前查看的是 `JSON` 或 `智能文本` 页签而不是 `正向`。

处理方式：

- 运行节点一次。
- 刷新浏览器。
- 切换到 `正向` 页签。
- 打开 `历史` 查看是否有记录。

### 18.8 古风、商业、赛博互相串词

减少串词的方法：

- 每次只保留一个主风格轨道。
- 清理旧标签。
- 不要把多个强风格同时写入自定义补充。
- 随机模式选择 `重写主体与场景` 时，允许节点替换旧主体、场景和服装主线。

### 18.9 生成内容总是粉色、总是全身、总是某个物品

检查是否存在：

- `玫瑰粉调`、`高饱和` 等固定光影标签。
- `全景全身`、`人物完整入镜` 被锁定。
- 某个物品在白名单或自定义补充里。
- API 额外要求里写死了同一描述。

### 18.10 一张图出现上下重复、多个相同人物或多张脸

更新后先重新运行节点，确认 `推荐负面词` 中出现分屏、拼贴、故事板、堆叠肖像、重复脸、额外头部或克隆主体等对应词。普通单人正向提示词应包含“单张连续画面”“同一人物只出现一次”；双人/群像应包含“每人只出现一次”；角色设定图应显示跨视图身份一致性规则。

如果仍出现：

- 清理旧的 `纵向叙事分镜`、`360度环绕`、`快速场景切换`、`先推近再拉远` 或“分别展示多个阶段”等自定义标签。旧工作流中的已知运镜会自动改写，但自由文本仍应手动整理。
- 单人图不要同时选择双人、群像、镜中倒影、四联画或角色三视图；确实需要双人时明确选情侣/伴侣与双人同框，不要只在长句里含糊写“他们”。
- 检查下游是否又拼接了其他正向提示词，或是否使用了会强制分镜、漫画页、角色表的 LoRA、ControlNet、工作流模板。
- 负向提示词必须连接到实际采样器使用的负向编码输入；只在节点终端里看到推荐负面词但没有接到下游，不会影响生图。
- 竖幅超长比例更容易放大上下分镜倾向。合同和负向词已做抑制，但模型/LoRA 本身若强偏角色表，可先换常规基础模型验证。

### 18.11 视频提示词没有经过模型润色

先看 `已选标签` 或 `JSON结果` 中的 `视频提示词模型状态`：

- `未调用（仅Skill）`：当前模型来源是 `仅Skill`，这是预期行为，不是错误。
- `已采用模型润色`：模型返回的正文通过了长度、因果、语言和锚点校验。
- `模型调用成功，保留 Skill 结果`：模型有响应，但没有产生比 Skill 更可靠的变化。
- `模型调用失败或候选不合格，保留 Skill 结果`：请求失败、响应为空、正文含分析或模型改掉了主体/场景/动作；Skill 结果仍然可直接使用。

如果希望模型参与视频输出，请把 `模型来源` 切到 `本地模型` 或 `API接口`，然后确认模型对象支持文本调用接口。API 和完整长文模型建议把 `最大生成token` 保持在 `1800` 以上；内置 Qwen3.5 纯中文视频后置会自动使用最多 640 token 的短增量，无需单独拉高。视频模型调用失败不会影响前 7 个输出口，也不会清空第 8 个视频提示词。

## 19. 推荐工作流

### 19.1 稳定写实人像

1. 模板风格选 `真实感` 或 `商业摄影`。
2. 构图选 `全景全身` 或 `中景半身`。
3. 光影选择一到两个，不要太多。
4. 运行一次，看终端预览。
5. 不满意再用智能文本补一句方向。

### 19.2 批量随机找灵感

1. 打开 `运行时随机标签`。
2. 随机主题池选 `自动`。
3. 随机强度选 `中` 或 `强`。
4. 核心标签锁定数量不要太高。
5. 每次运行后到历史里查看和收藏。

### 19.3 智能文本精修

1. 在智能文本输入一句自然描述。
2. 点击 `匹配启用`。
3. 查看解析结果。
4. 如果偏离你的输入，切 `文本优先`。
5. 如果想保留节点标签，切 `节点优先`。

### 19.4 API 精修长提示词

1. 模型来源选 `API接口`。
2. 选择服务商，并优先填写 `env:环境变量名`；自定义地址按上面的可信来源规则配置。
3. 详细度选择 `详细`。
4. 输入智能文本或导入标签块编排。
5. 运行后检查终端预览和历史查看。

### 19.5 NSFW 工作台

1. 打开 `NSFW`。
2. 选择结构、镜头、负面和质量档位。
3. 可在 `自定义字段库` 添加自己的字段选项。
4. 点击 `启用工作台`。
5. 点击 `写回当前节点`。
6. 回到外面运行节点。

## 20. 给别人安装前的检查清单

分享前建议确认：

- 已清空直接填写的 API Key，或确认工作流里只保留安全的 `env:变量名` 引用。
- 节点文件夹完整。
- `web` 文件夹存在。
- `使用说明书.md` 存在。
- `一键检查依赖.bat` 和 `自动补装依赖.bat` 存在。
- 在自己的 ComfyUI 里能看到底部 UI 面板。
- 运行一次能输出 `首条正向提示词`。
- 历史里能查看本次生成提示词。
- NSFW 工作台启用后外部按钮状态正常。

## 21. 最短口诀

新手用：

```text
仅Skill + 标签 + 智能文本 + 终端预览
```

想更智能：

```text
API接口 + 智能文本 + 标签块编排
```

想更随机：

```text
运行时随机 + 自动主题池 + 中/强随机
```

想更可控：

```text
锁核心标签 + 降低随机强度 + 使用预设
```

想排错：

```text
看终端预览 -> 看历史 -> 看 ComfyUI 日志 -> 检查依赖
```

这个节点的核心思路是：标签负责素材，Skill 负责判断，模型/API 负责重写，终端预览和历史负责确认结果。只要这四件事能串起来，提示词就会越来越稳。
