# 真男人提示词阶段生成器

`ComfyUI-RealMan-Prompt-Stage-Generator` 是一个面向 ComfyUI 的阶段式提示词生成节点。它将标签选择、随机主题、模板风格、智能文本、Skill、本地/API 模型、设定图反推、标签块编排、NSFW 工作台、历史记录和严格去重整合到同一节点中。

## 主要功能

- 高级按钮直接读取后端完整目录，提供 90+ 模板风格与 110+ 随机主题方向；长目录使用下拉选择，不再被前端旧短名单截断，并补齐风格隔离、智能文本优先、极限随机和提示词优先等控制。
- 所有模式统一输出带剧情的自然语言提示词；每条都会形成“处境与动机、事件触发、主体回应、局势升级、情绪转折、环境与光线反馈、视觉高潮、开放定格”的视觉链路。中文成品统一为 `800-1200` 字，英文目标为 `420-560 words`。主提示词、智能文本、标签块、角色设定图、仅 Skill、本地模型、API 和模型后置共用同一剧情合同；模型短草稿会融入 Skill 剧情骨架，思考/分析包装会先清洗，只有确实不可用的输出才回退。
- 全局画面结构合同会把剧情收束到一张图里的一个决定性时刻：默认同一人物只出现一次且只有一个清晰头部和一张脸；显式双人/群像保留用户人数，但每人只出现一次并保持独立站位；角色设定图、三视图、四联画和显式镜中视图保留多视图结构。正向提示词与推荐负面词都会抑制上下重复、分屏、拼贴、故事板、堆叠肖像、克隆人物、重复脸和时间切片。
- 仅 Skill、本地模型和 API 三种生成路径。本地模型既支持节点内置 GGUF，也支持 `qwen模型`/通用模型输入连接的兼容对象；Ollama、LM Studio 等本地服务通过 API 面板调用。旧工作流的 `本地GGUF` 会自动迁移为 `本地模型`。
- 参考图片反推与角色设定图工作流。
- 标签块编排、用户标签库和预设/历史管理。
- 本机 Edge/Chromium 实时内嵌网页浏览、备用完整浏览器与显式提示词联网搜索；完整提示词可回填当前节点、复制，或拆分后加入标签库。
- 普通标签库现有 2498 个内置标签，新增丁达尔光效、体积光、光源互动、大气光学、天气、时段、天空天象、地质奇观、植被生态、建筑风景、山地、水域海岸与单帧稳定构图等目录；可选 NSFW 工作台提供 26 个结构化字段、850 个内置选项条目、16 套预设和 7 套负面方案，并支持预览、写回与负面提示词同步。
- 连续生成避重、最终提示词严格去重和缓存隔离。
- 主界面与紧凑工具栏两种前端交互模式。

### 性能与交互优化

- 节点元数据不再为 200 个隐藏槽位重复携带完整标签目录，添加节点、打开工作流和加载 `/object_info` 更轻量；旧工作流保存的标签字符串仍兼容。
- 槽位面板在第一次展开时才创建，每个分组的 20 个可搜索输入共用一份候选列表，收起状态不会生成数万项隐藏下拉 DOM。
- 标签库使用浏览器快照、压缩传输与 `ETag` 条件校验；标签没有增删时，标签、模型、预设和历史等常用面板直接复用已验证数据，不再反复等待整库下载。

### 标签容量与提示词逻辑

- 标签面板的 10 个分组现在每组提供 20 个可选槽位，旧工作流原有的前 10 个槽位名称保持不变；载入旧工作流时新增槽位自动填为“无”，不会挤乱后续设置。
- 标签弹窗会显示每组 `已选/上限`，达到上限后保留已选标签的取消能力，并把溢出标签明确放入“自定义补充”而不是静默丢弃。槽位面板、预设、NSFW 映射、随机器和状态归一化使用同一份容量配置，内部仍有 32 个槽位硬保护。
- 扩容不会让随机模式无边界地堆标签：随机自动补充最多沿用每组前 10 个自动槽位的预算，弱/中/强分别补 1/2/剩余容量；新增槽位主要用于人工选择、预设和 NSFW 映射。提示词仍使用 32 个自然语言片段预算，并按主体、场景、构图和安全约束优先级收敛，避免把 200 个可选槽位直接拼成标签串。
- 风格隔离作用于全部生成模式，与“运行时随机”是否开启无关。显式模板拥有主媒介；自动模板配合具体主题池时，由主题池档案确定一个主轨道。严格模式清除所有外来媒介词，平衡模式最多保留一个兼容质感桥，允许漂移模式最多保留一个有意的次风格。
- 剧情规划中的开端、升级和结尾是因果推理，不是要求绘图模型同时画出多个时间点。成品只显示当前动作峰值，过去和未来由衣料余势、道具状态、视线、光线和环境痕迹暗示，因此 800-1200 字不会再自动等同于上下分镜。

## 参数快速开始

节点默认配置适合直接使用：`仅Skill / 生成 3 条 / 纯中文 / 标准 / 完整结果 / 平衡收敛`。随机模式默认值为 `自动判断`，但 `运行时随机标签` 默认关闭；内置 GGUF 路线默认上下文 `8192`、GPU 层数 `-1`；API 默认超时 `120` 秒；模型采样默认 `1800 token / 温度 0.62 / top_p 0.9 / top_k 40 / 重复惩罚 1.08`。

| 用途 | 建议配置 |
| --- | --- |
| 快速稳定 | 仅 Skill、数量 `1-3`、标准、随机关闭 |
| API 长剧情 | 数量 `1-3`、详细、token `1800-2600`、温度 `0.62-0.78`、超时 `120-180` |
| 高差异批量 | 数量 `4-8`、随机开启、强/极限、锁定 `0-3`、seed `0` |
| 本地低内存 | 上下文 `4096-8192`、KV q8、数量 `1`、图片边长 `640-768` |
| 严格系列图 | 具体模板和主题池、严格风格隔离、节点优先、固定正数 seed |

人数与画面结构不需要额外参数：普通人物标签默认按单人单帧处理；选择 `双人互动`、情侣、伴侣或群像标签会自动切到多人同帧；只有启用 `设定图` 或显式选择三视图、四联画、镜中倒影时才允许多视图。不要在普通单图的 `额外要求` 中写“先……随后……最后分别展示”或“纵向分镜”。

用户预设现在会保存风格、随机、智能文本、图片反推、本地模型和采样参数，便于复现；API 密钥、额外请求头和运行缓存不会进入预设。所有参数的默认值、范围、显存档位、联动关系与完整配置方案见 [使用说明书：参数设置完整手册](./使用说明书.md#16-参数设置完整手册)。

## 节点兼容性

- 显示名称：`真男人提示词阶段生成器`
- 内部节点类型：`QwenTE_StagePromptGenerator`

内部类型保持不变，因此旧工作流仍可继续加载。

主面板会按节点确认是否真正创建成功，而不是只看前端脚本是否已经加载。遇到旧版或特殊 ComfyUI 前端暂时无法创建完整面板时，节点会自动显示 `TE MINI CONSOLE` 备用工具栏，并持续收起后来插入的 11-20 等原始槽位和高级参数；主面板后续重试成功后，备用栏会自动退出。主脚本不再使用 Safari/旧 WebView 容易拒绝的正则反向断言，标签目录接口暂时不可用时也会先从原生节点槽位建立最低可用主面板。需要主动使用兼容模式时，可在 ComfyUI 地址后加入 `?qwen_te_mini=1` 并刷新。

如果仍只看到大量原始参数，请确认 `custom_nodes` 中只保留一个启用的本插件目录。GitHub ZIP 解压得到的 `ComfyUI-RealMan-Prompt-Stage-Generator-main` 可以运行，但不要再同时保留旧目录、复制目录或另一个 Manager 安装；清理重复安装后完全重启 ComfyUI，再按 `Ctrl+F5`。主界面和备用工具栏脚本对标准目录、旧目录以及常见的 `-main` / `-master` 目录名都使用无缓存响应。

## 模型 API 状态

点击节点底部 `模型` 后，必须把 `模型来源` 切换为 `API`；停留在 `Skill` 时不会发送任何 API 请求。切换到不同服务商会写入该服务商的预设地址、模型和 `env:` 密钥引用，并清空旧的额外请求头，避免把认证信息带到其他服务商。切换到 `OpenAI兼容` 或 `自定义` 后，地址、模型和密钥需要重新填写。

`env:变量名` 通常只读取指定变量。兼容旧工作流时，`env:QWEN_TE_API_KEY` 为空会继续尝试当前服务商的标准变量，例如 `OPENAI_API_KEY` 或 `DEEPSEEK_API_KEY`。密钥留空时，命名远程服务商也会检查其预设环境变量；兼容、自定义和本地接口不会自动借用通用密钥。

`API地址` 只接受规范的 `http://` 或 `https://` Base URL，不得包含账号密码、查询参数、片段、反斜杠、转义点段、中文路径或控制字符。`API额外请求头` 支持 JSON 对象或每行 `名称: 值` / `名称=值`；格式错误、重复名称、空值和控制字符会直接报错，认证、Host、Content-Type、Content-Length、Cookie 等受保护字段不能覆盖。

模型面板会显示最近一次运行的 `未启用 / 待调用 / 调用成功 / 部分回退 / 调用已恢复 / 调用失败` 状态。JSON 中 `model_active_fallback_count` 表示最终仍使用 Skill 的输出条数；`model_call_failure_count` 记录历史失败次数，两者含义不同。配置签名同时覆盖服务商、地址、模型、Key 引用和额外 Header，但只输出不可逆摘要，不输出明文认证信息。

当前主要预设包括 DeepSeek `deepseek-v4-flash`、Groq `openai/gpt-oss-20b`、Claude `claude-haiku-4-5`、Gemini `gemini-2.5-flash`。OpenAI o1/o3/o4/GPT-5 会自动使用兼容的消息角色和 token 参数；DeepSeek V4 默认关闭 thinking；Claude/Gemini 原生接口会映射各自的采样和停止序列字段。API 默认超时为 120 秒；超时、断连、429 和服务端 5xx 会有界重试一次，认证错误与普通 4xx 不重试。用户填写的最大 token 作为每条上限生效，批量请求按条数扩展并封顶 8192，不再静默强制单条至少 1280。批量模型没有按分隔协议返回时，最多 8 条的小批量会自动逐条重试。

## 网页浏览器与提示词搜索

点击终端预览标题栏中的 `浏览器`。弹窗默认打开 `网页浏览器` 页签，可输入 `example.com`、完整 HTTPS 地址或普通搜索词，再点击 `打开`。弹窗打开时只显示本地主页，不会自动访问外部网站。

主浏览区不是 iframe。后端会启动隔离的本机 Microsoft Edge/Chrome/Chromium 无头会话，把实时网页画面传回节点，并把鼠标点击、拖动、滚轮、常用按键、粘贴和中文输入发送到真实浏览器页面。因此 GitHub、Google、百度等拒绝 iframe 的网站也不再受 `X-Frame-Options` 或 CSP 嵌入限制，返回、前进和刷新读取的是真实浏览器历史。交互期间使用低延迟原生尺寸画面，页面稳定后按设备像素比生成高清帧；拖动和滚轮只保留最新待发送事件，静止页面自动降低刷新频率。窗口尺寸或显示缩放变化后，网页视口和点击坐标会自动同步。

内嵌会话默认禁止下载，关闭弹窗或返回本地主页时会关闭对应浏览器进程并清理临时会话目录。若当前 Edge/Chromium 被系统策略禁止远程调试，节点会自动诊断失败原因，并用当前网址打开真正的独立可见浏览器窗口，不再停留在灰色失败页；之后点击 `打开` 会继续在这个独立浏览器模式中打开网址。独立窗口支持标签页、下载、登录、证书检查、系统选择器和复杂弹窗，右上角的 `独立浏览器` 按钮也可随时打开当前网址。地址栏右侧箭头仍用于普通新标签页打开当前网址。

入口会拒绝危险协议、带账号密码的网址以及直接输入的本机/内网地址；请求接口还要求本机同源和专用请求头。这只是入口保护，不是完整网络隔离，网页打开后仍可自行跳转或发起网络请求。

Windows 会自动查找 Microsoft Edge、Google Chrome 或 Chromium。需要指定便携 Chromium 时，可在启动 ComfyUI 前设置 `QWEN_TE_BROWSER_EXE` 为可信浏览器可执行文件的绝对路径；只接受 `msedge.exe`、`chrome.exe` 或 `chromium.exe`。插件更新后新增的本机浏览器接口需要完整重启 ComfyUI 才会生效。

切换到 `提示词搜索` 页签，输入画面主题或风格关键词，再点击 `搜索`。节点会从公开提示词来源检索完整提示词，并在在线来源不可用时使用本地标签库组合提示词。

提示词搜索结果不会自动参与生成。选择完整提示词后可执行 `回填提示词`、复制提示词，或明确执行拆分入库。网页只在手动打开后访问；提示词关键词只在点击 `搜索` 后发送到第三方来源，普通生成和仅 Skill 模式不会自动联网。

要启用通用网页搜索，请打开现有的 `online_search_config.json`，只修改其中的 SearXNG 地址和公开回退开关，不要用下面的片段覆盖整个文件：

```text
"searxng_base_url": "https://search.example.com",
"searxng_timeout_seconds": 8,
"public_source_fallback_enabled": false
```

也可在启动 ComfyUI 前设置环境变量 `QWEN_TE_SEARXNG_URL` 临时覆盖配置。SearXNG 返回有效结果后不会继续访问 Civitai 或 Lexica。将 `public_source_fallback_enabled` 设为 `false` 后，SearXNG 不可用或无结果时也不会访问公开来源，而是直接使用本地标签库回退；未配置 SearXNG 时该开关同样适用。

相同关键词的原始搜索样本会短时缓存，重复点击不会反复访问外部来源；标签是否已入库仍会按当前标签库重新计算。成功结果缓存约 3 分钟，全源失败只短暂缓存约 20 秒，网络恢复后无需长时间等待缓存失效。

SearXNG 实例必须启用 JSON 输出（`format=json`），当前只接受无账号信息的 HTTPS 地址，也不发送自定义认证头。本机自签名或企业 CA 可通过 `SSL_CERT_FILE` 或 `REQUESTS_CA_BUNDLE` 指定证书包；纯 HTTP 的 localhost 实例不会连接。公开仓库建议使用环境变量保存私有实例地址，避免误提交内部域名。
