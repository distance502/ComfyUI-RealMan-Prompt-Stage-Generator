# 随机主题池五模式优化设计

## 目标

在同一随机主题池下，同时优化 `真实感 / 插画感 / CG感 / 古风 / 神话感` 五种模板风格模式，让出图在保持同一主题骨架稳定的前提下，风格分化更明显。

## 成功标准

- 同一随机主题池下，`主体 / 主场景 / 主构图` 不被运行时随机漂移破坏。
- 五种模板风格各自保有可识别的风格核心，不会被跨媒介语义污染。
- 随机变化主要落在次级层：动作姿态、光影氛围、服装/材质细节、技术画质。
- `stage-only` 与完整出图链路都能复现上述行为。

## 方案

采用“模式优先锚定”方案：

1. 固定三层主题骨架
- 主体
- 主场景
- 主构图

2. 为五种模板风格分别定义更强的 `mode core`
- `真实感`：摄影、镜头、肤感、成片语义优先
- `插画感`：手绘、动画、绘本、媒介笔触优先
- `CG感`：渲染、材质、空间透视、角色卡语义优先
- `古风`：东方服饰、古典场景、卷轴/庭院构图优先
- `神话感`：神殿、云海、仪式性、奇观叙事优先

3. 收紧运行时随机的可漂移范围
- 允许变化：动作姿态、光影氛围、次级服装/材质、技术画质
- 限制变化：主体身份、主场景、主构图、模式核心

4. 增加跨模式污染约束
- `古风 / 神话感` 不回流现代摄影棚、工业、都市夜场语义
- `真实感` 不被 OVA / 赛璐璐 / 水彩纸媒介主导
- `插画感` 不被 raw photo / 纪实抓拍 / 真实棚拍主导
- `CG感` 不被水彩 / 古风庭院 / 神殿圣所主导

## 代码范围

- `stage_prompt/randomizer.py`
- `stage_prompt/prompt_builder.py`
- `stage_prompt/state_builder.py`
- `stage_prompt_generator.py`
- `tests/test_stage_prompt_modules.py`
- 必要时扩展 stage-only / 完整 comparison 脚本

## 验证路径

1. 本地单元与回归测试
2. 五模式 `stage-only` 对照
3. 五模式完整 `prompt + 出图` 对照

## 非目标

- 不新增 UI 参数面
- 不重写整个随机主题池体系
- 不为了强行分化而让题材主线漂移
